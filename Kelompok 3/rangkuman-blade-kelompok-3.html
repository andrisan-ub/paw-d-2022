REVIEW MATERI

INTRODUCTION


Blade merupakan sebuah template sederhana yang terdapat pada Laravel. File template blade menggunakan ekstensi file .blade.php dan pada umumnya disimpan di direktori resources/views.


Route::get('/', function () {
        return view('greeting', ['name' => 'anya']);
      });

DISPLAYING DATA


Data yang dikirim kepada view blade dapat ditampilkan dengan sintaks {{ }}. Misalnya, jika pada route terdapat kode: 


Route::get('/', function () {
      return view('welcome', ['nama' => 'Budi']);
  });

Maka data nama dapat ditampilkan sebagai berikut:


Halo, {{ $nama }}.

Sintaks {{ }} berfungsi seperti echo pada PHP, sehingga dapat pula diisi dengan fungsi.


Waktu UNIX timestamp sekarang: {{ time() }}.

#HTML Entity Encoding

Secara default, Blade akan melakukan double encoding pada HTML entity. untuk mencegah hal ini, dapat dilakukan dengan memanggil Blade::withoutDoubleEncoding pada method boot pada file AppServiceProvider. 


class AppServiceProvider extends ServiceProvider
  {
      ...
      public function boot()
      {
          Blade::withoutDoubleEncoding();
      }
  }

#Displaying Unescaped Data

Secara default, sintaks {{ }} akan dikirim secara otomatis ke htmlspecialchars pad PHP untuk mencegah serangan XSS dan injeksi. Untuk mencegah perlindungan ini, dapat menggunakan sintaks {!! !!}. 


{!! $var !!}.

#Blade & JavaScript Frameworks

Beberapa framework JavaScript juga menggunakan sintaks kurung kurawal. Untuk mencegah Blade melakukan interpret, dapat menggunakan tanda @ di depan kurung. Blade tidak akan menyentuh bagian kode tersebut sehingga hasil outputnya dapat digunakan pada framework lain. 


Halo, @{{ $nama }}

  <!-- HTML output -->
  Halo, {{ $nama }}

Tanda @ juga dapat digunakan pada directive.


{{-- Blade template --}}
  @@if()
  @@for()
                             
  <!-- HTML output -->
  @if()
  @for()

#@verbatim Directive

Kode di dalam @verbatim tidak akan diinterpretasi oleh Blade, sehingga tidak perlu menambahkan tanda @ pada setiap variable. 


@verbatim
      <div class="container">
          Hello, {{ name }}.
      </div>
  @endverbatim

BLADE DIRECTIVES


#If statement

Percabangan dilakukan dengan menggunakan @if, @elseif, @else, dan @endif


@if ($var > 10)
      Variable dibawah 10
  @elseif ($var > 100)
      Variable dibawah 100
  @else
      Variable berukuran besar
  @endif

Kode pada @unless dijalankan jika kondisi bernilai false.


@unless (Auth::check())
    Anda belum login
  @endunless

@iseet digunakan untk mengecek apakah variable sudah diinisiasi. @empty akan mengecek jika variable bernilai kosong. 


@isset($records)
    Variable record tidak null
  @endisset
                         
  @empty($records)
    Variable record kosong
  @endempty

@auth digunakan untuk mengecek jika pengguna sudah terautentikasi, sedangkan @guest mengecek jika pengguna belum terautentikasi. Jika perlu dapat dilakukan pengecekan pengguna terautentikasi sebagai apa. 


@auth
    Pengguna teruatentikasi
  @endauth

  @guest
    Pengguna belum terautentikasi
  @endguest
  
  @auth('guest')
    Terautentikasi sebagai admin
  @endauth

  @guest('admin')
    Pengguna bukan admin
  @endguest

#Switch Statements

Switch dilakukan dengan menggunakan @switch, @case, @default, @break, dan @endswitch


@switch($var)
      @case(1)
          ...
          @break
                         
      @case(2)
          ...
          @break
                         
      @default
          ...
  @endswitch

#Loops

Perulangan pada blade menggunakan @for, @endfor, @foreach, @endforeach, @while, dan @endwhile. Directive @continue dan @break dapat digunakan pada perulangan sebagai mana pada PHP. 


@for ($i = 0; $i < 10; $i++)
      i: {{ $i }}
  @endfor
 
  @foreach ($users as $user)
      Id pengguna: {{ $user->id }}
  @endforeach

  @while ($i < 10)
      i: {{ $i }}
  @endwhile

#The Loop Variable

untuk mendeklarasikan sebuah foreach loop, tidak perlu menggunakan syntax php, hanya menggunakan fungsi @foreach, kemudian menambahkan kurung kurawal {} untuk membuka dan menutup php echo statement, lalu diakhiri dengan @endforeach.


@foreach ($users as $user)
        @if ($loop->first)
            This is the first iteration.
        @endif
                         
        @if ($loop->last)
            This is the last iteration.
        @endif        
     This is user {{ $user->id }}
     @endforeach

Jika berada dalam nested loop, kita dapat mengakses parent's loop variabel melalui properti dari parent.


@foreach ($users as $user)
        @foreach ($user->posts as $post)
            @if ($loop->parent->first)
                This is the first iteration of the parent loop.
            @endif
        @endforeach
     @endforeach

#Conditional Classes

Direktif @class mengkompilasi sebuah string class CSS secara kondisional. Direktif menerima class array di mana array key berisi sebuah class atau atau class baru dengan menggunakan ekspresi boolean untuk mendapatkan nilai.


@php
        $isActive = false;
        $hasError = true;
     @endphp
                         
     <span @class(['p-4', 'font-bold' => $isActive, 'text-gray-500' => ! $isActive, 'bg-red' => $hasError,])></span>
     <span class="p-4 text-gray-500 bg-red"></span>

#Additional Attributes

Direktif @checked dapat digunakan untuk menunjukkan apakah input pada chechbox HTML telah diperiksa. Direktif tersbeut akan mengeluarkan output jika kondisi bernilai "true". 


<input type="checkbox"
        name="active"
        value="active"
     @checked(old('active', $user->active)) />

Perintah @selected dapat digunakan untuk menunjukkan apakah opsi yang diberikan harus "dipilih" 


<select name="version">
        @foreach ($product->versions as $version)
            <option value="{{ $version }}" @selected(old('version') == $version)>
                {{ $version }}
            </option>
        @endforeach
     </select>

Perintah @disabled dapat digunakan untuk menunjukkan apakah ada elemen tertentu yang harus "dinonaktifkan" 


<button type="submit" @disabled($errors->isNotEmpty())>Submit</button>

Perintah @readonly dapat digunakan untuk menunjukkan apakah elemen yang diberikan hanya dapat dibaca. 


<input type="email"
        name="email"
        value="email@laravel.com"
        @readonly($user->isNotAdmin()) />

Perintah @required dapat digunakan untuk menunjukkan apakah elemen tertentu wajib diisi. 


<input type="text"
        name="title"
        value="title"
        @required($user->isAdmin()) />

#Including Subviews

Direktif @include memungkinkan untuk menyisipkan sebuah blade's view ke blade's view yang lain. Semua variabel yang terdapat pada blade's view parent akan tersedia juga pada blade's view child. Atau dalam istilah lain, biasa disebut inheritance. 


<div>
        @include('shared.errors')
                         
     <form>
        // Konten
     </form>
    </div>

Meskipun tampilan pada child akan mewarisi semua data pada tampilan parent, kita tetap dapat menambahkan array yang berisi data tambahan untuk tampilan child.


@include('view.name', ['status' => 'complete'])

Jika menggunakan direktif @include pada view yang tidak ada, maka laravel akan throw an error. Solusinya adalah menggunakan direktif @includeif 


@includeIf('view.name', ['status' => 'complete'])

Jika ingin menggunakan direktif @include pada ekspresi boolean yang manghasilkan nilai salah atau benar, dapat menggunakan direktif @includeWhen dan @includeUnless 


@includeWhen($boolean, 'view.name', ['status' => 'complete'])
 
      @includeUnless($boolean, 'view.name', ['status' => 'complete'])

Jika ingin menyertakan tampilan pertama dari view tertentu, dapat menggunakan perintah @includefirst 


@includeFirst(['custom.admin', 'admin'], ['status' => 'complete'])

#The @Once Directive

Direktif @once memungkinkan untuk menentukan bagian dari template yang akan dievaluasi dalam sekali rendering. Hal tersebut berguna untuk melakukan push JavaScript ke header halaman menggunakan stack. 


@once
        @push('scripts')
            <script>
                // KODE
            </script>
        @endpush
      @endonce

Karena direktif @once sering digunakan bersama dengan direktif @push atau @prepend, dapat juga menggunakan @pushOnce dan @prependOnce


@pushOnce('scripts')
        <script>
            // KODE
        </script>
      @endPushOnce

#Raw PHP

Di beberapa situasi, sangat berguna untuk mengaitkan kode PHP pada views. KIta bisa menggunakan Blade @php direktif untuk mengeksekusi blok PHP biasa. 


        
        @php
            $counter = 1;
        @endphp

Cara penulisan satu statement PHP


        
        @php($counter = 1)

#Comments

Pada blade juga menyediakan komentar dalam tampilannya, tetapi komentar tersebut tidak akan includ pada tampilan HTML return. 


{{-- This comment will not be present in the rendered HTML --}}

#Scoped Slots

Scoped Slots memungkinkan untuk mengakses data atau metode dari komponen di dalam slot. Untuk mencapai perilaku serupa di Laravel dapat didefinisikan dengan metode atau properti publik pada komponen dan mengakses komponen di dalam slot melalui variabel $component. 



    <x-alert>
        <x-slot:title>
            {{ $component->formatAlert('Server Error') }}
        </x-slot>
                             
        <strong>Whoops!</strong> Something went wrong!
    </x-alert>

#Slot Attributes

Dapat menetapkan atribut tambahan ke slot



    <x-alert>
        <x-slot:title>
            {{ $component->formatAlert('Server Error') }}
        </x-slot>
                             
        <strong>Whoops!</strong> Something went wrong!
    </x-alert>

Untuk berinteraksi dengan atribut slot, dapat mengakses properti atribut dari variabel slot



    <x-card class="shadow-sm">
        <x-slot:heading class="font-bold">
            Heading
        </x-slot>

        Content
                             
        <x-slot:footer class="text-sm">
            Footer
        </x-slot> 
    </x-card>

#Inline Component Views

Untuk komponen yang sangat kecil dan rumit untuk mengelola kelas komponen dan template tampilan komponen, dapat mengembalikan markup komponen langsung dari render. 



    /**

    *
    * @return \Illuminate\View\View|\Closure|string
    */
    public function render()
    {
        return <<<'blade'

                {{ $slot }}
            </div>
        blade;
    }

COMPONENTS


#Rendering Components

Untuk menanpilkan komponen, kita bisa menggunakan tag komponen Blade. Tag ini diawali dengan string x-, diikuti dengan kebab case name dari class komponen: 


        
        <x-alert/>
 
        <x-user-profile/>

Jika class komponennya berada pada direktori app/View/Component, maka bisa menggunakan titik "." untuk mengindikasikan direktori bersarang. Sebagai contoh, diasumsikan lokasi komponen berada pada app/View/Components/Inputs/Button.php, maka bisa dituliskan : 


        
        <x-inputs.button>

Kita dapat meneruskan data ke komponen Blade menggunakan atribut HTML. Hard-coded, nilai primitif dapat diteruskan ke komponen menggunakan string atribut HTML sederhana. Ekspresi dan variabel PHP harus diteruskan ke komponen melalui atribut yang menggunakan karakter ":" sebagai prefix: 



    <x-alert type="error" :message="$message"/>

Kita harus mendefinisikan semua atribut data komponen dalam konstruktor classnya. Semua properti publik pada komponen akan secara otomatis tersedia untuk view komponen. Tidak perlu meneruskan data ke view dari metode render komponen: 



    <?php
 
    namespace App\View\Components;
     
    use Illuminate\View\Component;
     
    class Alert extends Component
    {
        /**
         * The alert type.
         *
         * @var string
         */
        public $type;
     
        /**
         * The alert message.
         *
         * @var string
         */
        public $message;
     
        /**
         * Create the component instance.
         *
         * @param  string  $type
         * @param  string  $message
         * @return void
         */
        public function __construct($type, $message)
        {
            $this->type = $type;
            $this->message = $message;
        }
     
        /**
         * Get the view / contents that represent the component.
         *
         * @return \Illuminate\View\View|\Closure|string
         */
        public function render()
        {
            return view('components.alert');
        }
    }

Ketika komponen sudah di-render, kit bisa menampilkan konten dari variabel komponen publik dengan meng-echo nama variable: 



    <div class="alert alert-{{ $type }}">
         {{ $message }}
    </div>

#Casing


Argumen konstruktor komponen harus ditentukan menggunakan camelCase, sedangkan kebab-case harus digunakan saat mereferensikan nama argumen dalam atribut HTML Anda. Misalnya, diberikan konstruktor komponen berikut: 



    /**
    * Create the component instance.
    *
    * @param  string  $alertType
    * @return void
    */
   public function __construct($alertType)
   {
       $this->alertType = $alertType;
   }

Argumen $alertType dapat diberikan ke komponen seperti: 



    <x-alert alert-type="danger"/>

#Short Attribute Syntax


Saat meneruskan atribut ke komponen, kita juga dapat menggunakan sintaks "short attribute": 



    {{-- Short attribute syntax... --}}
    <x-profile :$userId :$name />
     
    {{-- Is equivalent to... --}}
    <x-profile :user-id="$userId" :name="$name" />

#Escaping Attribute Rendering


Karena beberapa framework JavaScript seperti Alpine.js juga menggunakan atribut awalan titik dua, kita dapat menggunakan awalan titik dua (::) untuk memberi tahu Blade bahwa atribut tersebut bukan PHP. Misalnya, diberikan komponen berikut: 



    <x-button ::class="{ danger: isDeleting }">
          Submit
    </x-button>

HTML berikut akan dirender oleh Blade: 



    <button :class="{ danger: isDeleting }">
            Submit
    </button>

#Component Methods


Selain variabel publik yang tersedia untuk template komponen Anda, metode publik apa pun pada komponen dapat dipanggil. Misalnya, terdapat sebuah komponen yang memiliki method isSelected: 



    /**
    * Determine if the given option is the currently selected option.
    *
    * @param  string  $option
    * @return bool
    */
    public function isSelected($option)
    {
        return $option === $this->selected;
    }

Kita dapat menjalankan method ini dari template komponen kita dengan memanggil variabel yang cocok dengan nama method: 



    <option {{ $isSelected($value) ? 'selected' : '' }}
    value="{{ $value }}">
        {{ $label }}
    </option>

#Accessing Attributes & Slots Within Component Classes


Komponen blade juga memungkinkan kita mengakses nama komponen, atribut, dan slot di dalam metode render class. Namun, untuk mengakses data ini, kita harus mengembalikan penutup dari metode render komponen kita. Penutup akan menerima array $data sebagai satu-satunya argumennya. Array ini akan berisi beberapa elemen yang memberikan informasi tentang komponen: 



    /**
    * Get the view / contents that represent the component.
    *
    * @return \Illuminate\View\View|\Closure|string
    */
   public function render()
   {
       return function (array $data) {
           // $data['componentName'];
           // $data['attributes'];
           // $data['slot'];
    
           return '<div>Components content</div>';
       };
   }

componentName sama dengan nama yang digunakan dalam tag HTML setelah awalan x-. Jadi <x-alert /> pada componentName akan diberi tanda. Elemen atribut akan berisi semua atribut yang ada pada tag HTML. Elemen slot adalah instance Illuminate\Support\HtmlString dengan konten slot komponen.

Penutup harus mengembalikan sebuah string. Jika string yang di-return sesuai dengan view yang ada, view tersebut akan dirender; jika tidak, string yang dikembalikan akan dievaluasi sebagai view Blade sebaris (inline).


#Additional Dependencies


Jika komponen kita memerlukan dependensi dari Laravel, maka dapat dicantumkan sebelum atribut data komponen mana pun dan secara otomatis akan dimasukkan oleh container: 



    use App\Services\AlertCreator;
 
    /**
     * Create the component instance.
     *
     * @param  \App\Services\AlertCreator  $creator
     * @param  string  $type
     * @param  string  $message
     * @return void
     */
    public function __construct(AlertCreator $creator, $type, $message)
    {
        $this->creator = $creator;
        $this->type = $type;
        $this->message = $message;
    }

#Hiding Attributes / Methods


Jika ingin mencegah beberapa metode atau properti publik diekspos sebagai variabel ke template komponen, kita dapat menambahkannya ke properti array $except pada komponen : 



    <?php
 
    namespace App\View\Components;
     
    use Illuminate\View\Component;
     
    class Alert extends Component
    {
        /**
         * The alert type.
         *
         * @var string
         */
        public $type;
     
        /**
         * The properties / methods that should not be exposed to the component template.
         *
         * @var array
         */
        protected $except = ['type'];
    }

#Component Attributes


Laravel telah memeriksa cara meneruskan atribut data ke komponen; namun, terkadang kita mungkin perlu menentukan atribut HTML tambahan, seperti class, yang bukan merupakan bagian dari data yang diperlukan agar komponen berfungsi. Biasanya, kita ingin meneruskan atribut tambahan ini ke elemen root template komponen. Misalnya, bayangkan kita ingin merender komponen alert seperti ini: 



    <x-alert type="error" :message="$message" class="mt-4"/>

Semua atribut yang bukan bagian dari konstruktor komponen akan secara otomatis ditambahkan ke "attribute bag" komponen. Attribute bag ini secara otomatis tersedia untuk komponen melalui variabel $attributes. Semua atribut dapat dirender dalam komponen dengan meng-echo variabel ini: 



    <div {{ $attributes }}>
         <!-- Component content -->

    </div>

#Default / Merged Attributes


Terkadang kita mungkin perlu menentukan nilai default untuk atribut atau menggabungkan nilai tambahan ke dalam beberapa atribut komponen. Untuk mencapai ini, kita dapat menggunakan metode penggabungan attribute bag. Method ini sangat berguna untuk mendefinisikan satu set CSS default yang harus selalu ada pada komponen: 



    <div {{ $attributes->merge(['class' => 'alert alert-'.$type]) }}>
         {{ $message }}
    </div>

Jika kita menganggap komponen ini digunakan seperti ini: 



    <x-alert type="error" :message="$message" class="mb-4"/>

Laravel telah memeriksa cara meneruskan atribut data ke komponen; namun, terkadang kita mungkin perlu menentukan atribut HTML tambahan, seperti class, yang bukan merupakan bagian dari data yang diperlukan agar komponen berfungsi. Biasanya, kita ingin meneruskan atribut tambahan ini ke elemen root template komponen. Misalnya, bayangkan kita ingin merender komponen alert seperti ini: 



    <div class="alert alert-error mb-4">
          <!-- Contents of the $message variable -->
    </div>

#Conditionally Merge Classes


Terkadang kita mungkin ingin menggabungkan class jika kondisi yang diberikan benar. Kita dapat melakukannya melalui metode class, yang menerima array class di mana kunci array berisi class atau class yang ingin kita tambahkan, sedangkan nilainya adalah ekspresi boolean. Jika elemen array memiliki kunci numerik, elemen tersebut akan selalu disertakan dalam daftar class yang dirender: 



    <div {{ $attributes->class(['p-4', 'bg-red' => $hasError]) }}>
            {{ $message }}
    </div>

Jika kita perlu menggabungkan atribut lain ke komponen kita, maka dapat menghubungkan metode penggabungan ke metode class: 



    <button {{ $attributes->class(['p-4'])->merge(['type' => 'button']) }}>
         {{ $slot }}
    </button>

#Non-Class Attribute Merging


Saat menggabungkan atribut yang bukan atribut class, nilai yang diberikan ke metode gabungan akan dianggap sebagai nilai "default" dari atribut tersebut. Namun, tidak seperti atribut class, atribut ini tidak akan digabungkan dengan nilai atribut yang dimasukkan. Sebaliknya, mereka akan ditimpa. Misalnya, implementasi komponen tombol mungkin terlihat seperti berikut: 



    <button {{ $attributes->merge(['type' => 'button']) }}>
         {{ $slot }}
    </button>

Untuk merender komponen tombol dengan tipe custom, mungkin ditentukan saat menggunakan komponen. Jika tidak ada tipe yang ditentukan, tipe tombol akan digunakan: 



    <x-button type="submit">
          Submit
    </x-button>

HTML yang dirender dari komponen tombol dalam contoh ini adalah: 



    <x-button type="submit">
          Submit
    </x-button>

Jika kita ingin atribut selain kelas memiliki nilai default dan nilai yang dimasukkan digabungkan, kita dapat menggunakan metode prepends. Dalam contoh ini, atribut data-controller akan selalu dimulai dengan profile-controller dan nilai data-controller tambahan apa pun yang dimasukkan akan ditempatkan setelah nilai default ini: 



    <div {{ $attributes->merge(['data-controller' => $attributes->prepends('profile-controller')]) }}>
            {{ $slot }}
    </div>

#Retrieving & Filtering Attributes


Kita dapat memfilter atribut menggunakan metode filter. Metode ini menerima penutup yang seharusnya mengembalikan nilai true jika kita ingin mempertahankan atribut di tas atribut: 



    {{ $attributes->filter(fn ($value, $key) => $key == 'foo') }}


Untuk kenyamanan, kita dapat menggunakan metode whereStartsWith untuk mengambil semua atribut yang kuncinya dimulai dengan string yang diberikan: 



    {{ $attributes->whereStartsWith('wire:model') }}

Sebaliknya, metode whereDoesntStartWith dapat digunakan untuk mengecualikan semua atribut yang kuncinya dimulai dengan string yang diberikan: 



    {{ $attributes->whereDoesntStartWith('wire:model') }}

Dengan menggunakan metode first, kita dapat merender atribut pertama dalam tas atribut yang diberikan: 



    {{ $attributes->whereStartsWith('wire:model')->first() }}

Jika kita ingin memeriksa apakah ada atribut pada komponen, kita dapat menggunakan metode has. Metode ini menerima nama atribut sebagai satu-satunya argumennya dan mengembalikan boolean yang menunjukkan apakah atribut tersebut ada atau tidak: 



    @if ($attributes->has('class'))
         <div>Class attribute is present</div>
    @endif

kita dapat mengambil nilai atribut tertentu menggunakan metode get: 



    {{ $attributes->get('class') }}

ANONYMOUS COMPONENTS


#Anonymous Index Components

Ketika sebuah komponen terdiri dari banyak template Blade, saat ingin mengelompokkan template komponen yang diberikan dalam satu direktori.


Misalnya, bayangkan komponen "akordeon" dengan struktur direktori berikut:

/resources/views/components/accordion.blade.php
/resources/views/components/accordion/item.blade.php

Struktur direktori ini memungkinkan untuk merender komponen akordeon dan itemnya seperti:

<x-accordion>
<x-accordion.item>
<...>
</x-accordion.item>
</x-accordion>

Namun, untuk merender komponen akordeon melalui x-accordion, kami terpaksa menempatkan template komponen akordeon "indeks" di direktori resources/views/components alih-alih menumpuknya di dalam direktori akordeon dengan templat terkait akordeon lainnya. Untungnya, Blade memungkinkan untuk menempatkan file index.blade.php di dalam direktori template komponen. Ketika template index.blade.php ada untuk komponen, itu akan dirender sebagai simpul "root" dari komponen. Jadi, kita dapat terus menggunakan sintaks Blade yang sama seperti yang diberikan pada contoh di atas. Namun, akan disesuaikan struktur direktori seperti berikut:

/resources/views/components/accordion/index.blade.php
/resources/views/components/accordion/item.blade.php

#Data Properties/Attributes

Karena komponen anonim tidak memiliki kelas terkait. Kita dapat menentukan atribut mana yang harus dianggap sebagai variabel data menggunakan arahan @props di bagian atas template Blade komponen Anda. Semua atribut lain pada komponen akan tersedia melalui tas atribut komponen. Jika ingin memberikan nilai default pada variabel data, Anda dapat menentukan nama variabel sebagai kunci array dan nilai default sebagai nilai array:


<!-- /resources/views/components/alert.blade.php -->
@props(['type' => 'info', 'message'])
<div {{ $attributes->merge(['class' => 'alert alert-'.$type]) }}>
{{ $message }}
</div>

Mengingat definisi komponen di atas, kami dapat membuat komponen seperti ini:

<x-alert type="error" :message="$message" class="mb-4"/>

#Accessing Parent Data

Terkadang Anda mungkin ingin mengakses data dari komponen induk di dalam komponen turunan. Dalam kasus ini, dapat menggunakan arahan @aware. Misalnya, bayangkan kita sedang membangun komponen menu kompleks yang terdiri dari parent x-menu dan child x-menu.item:


<x-menu color="purple">
<x-menu.item>...</x-menu.item>
<x-menu.item>...</x-menu.item>
</x-menu>

Komponen x-menu mungkin memiliki implementasi seperti berikut:

<!-- /resources/views/components/menu/index.blade.php -->
@props(['color' => 'gray'])
<ul {{ $attributes->merge(['class' => 'bg-'.$color.'-200']) }}>
 {{ $slot }}
</ul>

>>>>>>> Stashed changes

Karena prop warna hanya diteruskan ke parent x-menu, itu tidak akan tersedia di dalam x-menu.item. Namun, jika kita menggunakan direktif @aware, kita juga dapat membuatnya tersedia di dalam x-menu.item:

<!-- /resources/views/components/menu/item.blade.php -->
@aware(['color' => 'gray'])
<li {{ $attributes->merge(['class' => 'text-'.$color.'-800']) }}>
{{ $slot }}
</li>

Arahan @aware tidak dapat mengakses data induk yang tidak secara eksplisit diteruskan ke komponen induk melalui atribut HTML. Nilai @props default yang tidak secara eksplisit diteruskan ke komponen induk tidak dapat diakses oleh direktif @aware.

#Anonymous Component Namespaces

Metode anonymousComponentNamespace menerima "jalur" ke lokasi komponen anonim sebagai argumen pertama dan "namespace" yang komponennya harus ditempatkan sebagai argumen kedua. Seperti yang akan dilihat pada contoh di bawah, "namespace" akan diawali dengan nama komponen saat komponen dirender. Biasanya, metode ini harus dipanggil dari metode boot salah satu penyedia layanan aplikasi:


/**
                        * Bootstrap any application services.
                        *
                        * @return void
                        */
public function boot()
{
Blade::anonymousComponentNamespace('flights.bookings.components', 'flights');
}

Dengan contoh di atas, dapat merender komponen panel yang ada di dalam direktori komponen yang baru didaftarkan seperti:


<x-flights::panel :flight="$flight" />

BUILDING LAYOUTS


#Layouts Using Components

Sebagian besar aplikasi web mempertahankan tata letak umum yang sama di berbagai halaman. Akan sangat merepotkan dan sulit untuk memelihara aplikasi kita jika kita harus mengulang seluruh tata letak HTML di setiap tampilan yang kita buat. Untungnya, lebih mudah untuk mendefinisikan tata letak ini sebagai satu komponen Blade dan kemudian menggunakannya di seluruh aplikasi kita.


#Defining The Layout Component

<html>
<head>
<title>{{ $title ?? 'Todo Manager' }}</title>
</head>
<body>
<h1>Todos</h1>
</hr>
{{ $slot }}
</body>
</html>

#Applying The Layout Component

Setelah komponen tata letak ditentukan, kami dapat membuat tampilan Blade yang menggunakan komponen tersebut. Dalam contoh ini, kita akan mendefinisikan tampilan sederhana yang menampilkan daftar tugas kita:

<x-layout>
@foreach ($tasks as $task)
{{ $task }}
@endforeach
</x-layout>

Ingat, konten yang disuntikkan ke dalam komponen akan diberikan ke variabel $slot default di dalam komponen layout kita. Seperti yang mungkin telah Anda perhatikan, tata letak kami juga mengikuti slot $title jika tersedia. Jika tidak, judul default akan ditampilkan. Kami dapat menyuntikkan judul khusus dari tampilan daftar tugas kami menggunakan sintaks slot standar yang dibahas dalam dokumentasi komponen:

<x-layout>
<x-slot:title>
Custom Title
</x-slot>
@foreach ($tasks as $task)
{{ $task }}
@endforeach
</x-layout>

Sekarang setelah kita mendefinisikan tampilan tata letak dan daftar tugas, kita hanya perlu mengembalikan tampilan tugas dari rute:

use App\Models\Task;
Route::get('/tasks', function () {
return view('tasks', ['tasks' => Task::all()]);
});

#Layout Using Template Inheritance

Layout dapat dibuat dengan template inheritance dimana layout dapat diturunkan ke childnya. Agar child dapat menampilkan template yang diturunkan dari parentnya dapat menggunkana notasi @extends. Notasi @yield digunakan untuk mendefinisikan baris pada parent agar di child dapat disesuaikan. Notasi @section digunakan untuk mendefinisikan baris yang memiliki notasi @yield pada parentnya.


Syntax Parent Layout

<html>
<head>
<title>App Name - @yield('title')</title>
</head>
<body>
@section('sidebar')
This is the master sidebar.
@show
<div class="container">
@yield('content')
</div>
</body>
</html>

Syntax Child Layout

@extends('layouts.app')
@section('title', 'Page Title')
@section('sidebar')
@parent
<p>This is appended to the master sidebar.</p>
@endsection
@section('content')

#CSRF Field

CSRF token ini dibuat secara otomatis untuk setiap pengguna. Token ini berupa string acak yang dikelola oleh Laravel untuk memverifikasi permintaan user agar user terlindungi dari serangan CSRF (Cross-Site Request Forgery).


<form method="POST" action="/profile">
@csrf
...
</form>

#Method Field

HTML ini tidak dapat membuat request PUT, PATCH, atau DELETE sehingga perlu untuk menambahkan notasi @method. 


<form action="/foo/bar" method="POST">
@method('PUT')
...
</form>

#Validation Error

Notasi @error berfungsi untuk mengecek suatu atribut apabila terjadi error dapat menampilkan pesan error tersebut. Selain itu juga dapat menambahkan notasi @else yang memberikan tampilan jika tidak error dan diakhiri dengan notasi @enderror


<label for="title">Post Title</label>
<input id="title" type="text" class="@error('title') is-invalid @enderror">
@error('title')
<div class="alert alert-danger">{{ $message }}</div>
@enderror
