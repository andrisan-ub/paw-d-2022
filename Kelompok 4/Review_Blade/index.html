HTML Entity Encoding

Pengertian HTML Entity Encoding

Secara umum, Blade akan menggandakan encode entitas HTML. Jika ingin menonaktifkan penyandian ganda, panggil method “Blade::withoutDoubleEncoding” dari method “boot” pada class


Contoh Syntax

“AppServiceProvider”
<?php
 
namespace AppProviders;
 
use IlluminateSupportFacadesBlade;
use IlluminateSupportServiceProvider;
 
class AppServiceProvider extends ServiceProvider
{
   /**
    * Bootstrap any application services.
    *
    * @return void
    */
   public function boot()
   {
       Blade::withoutDoubleEncoding();
   }
}

Switch Statements

Penjelasan Switch Statements

Sama halnya saat kita menggunakan 'if', kita dapat menggunakan '@' saat kita ingin menggunakan 'switch'.
Contoh penggunaan: @switch, @case, @break, @default, dan @endswitch.


Contoh Syntax

@php
    $i = 2;
@endphp
@switch($i)
    @case(1)
        Ini case pertama
    @break

    @case(2)
        Ini case kedua
    @break

    @default
        Ini case default
@endswitch

Akan tampil "Ini case kedua"

Components

Rendering Components

Penjelasan Rendering Components

Untuk menampilkan sebuah komponen, kita dapat menggunakan tag <x-..> dan diikuti dengan nama dari component class, misalnya <x-alert/>.
Jika component class berada di dalam direktori app/View/Components, maka kita dapat menggunakan karakter ".", semisal sebuah komponen berlokasi di app/View/Components/Inputs/Button.php, kita dapat menggunakan <x-inputs.button/>.


Contoh Syntax

<x-alert/>
Menampilkan komponen dengan class "alert"

<x-inputs.button/>
Menampilkan komponen "button" pada app/View/Components

Passing Data To Components

Penjelasan Passing Data To Components

Kita dapat meneruskan data ke komponen Blade menggunakan atribut HTML. Nilai primitif dapat diteruskan ke komponen menggunakan string atribut HTML sederhana. Variabel PHP harus diteruskan ke komponen melalui atribut yang menggunakan karakter (:) sebagai awalan atau prefix.

Contoh Syntax

<x-alert type="error" :message="$message"/>


<?php
    namespace App\View\Components;
    use Illuminate\View\Component;
    class Alert extends Component {

/**
   * The alert type. 
   *
   * @var string
   */
   public $type;

/**
   * The alert message.
   *
   * @var string
   */
   public $message;

/**
   * Create the component istance
   *
   * @param  string  $type
   * @param  string  $message
   * @return void
   */

Kita harus mendefinisikan semua atribut data komponen dalam konstruktor kelasnya. Semua properti publik pada komponen akan secara otomatis tersedia untuk tampilan komponen. Tidak perlu meneruskan data ke tampilan dari metode render komponen.


Contoh Syntax

 $this->type = $type;

Anonymous Components Namespaces

Penjelasan Anonymous Components Namespaces

Komponen anonim biasanya ditentukan dengan menempatkan template Blade di dalam direktori resource/views/components Anda. Namun, terkadang Anda mungkin ingin mendaftarkan jalur komponen anonim lainnya dengan Laravel selain jalur default. Untuk memberi tahu Laravel tentang lokasi komponen anonim ini, Anda dapat menggunakan metode anonymousComponentNamespace yang disediakan oleh Blade.


Contoh Syntax 

/**
* Bootstrap any application services.
*
* @return void
*/

public function boot() {
Blade::anonymousComponentNamespace
    ('flights.bookings.components'
     'flights') }

Metode ini menerima jalur ke lokasi komponen anonim sebagai argumen pertama dan "namespace" yang komponennya harus ditempatkan sebagai argumen kedua. Seperti yang akan Anda lihat pada contoh di bawah, "namespace" akan diawali dengan nama komponen saat komponen dirender. Biasanya, metode ini harus dipanggil dari metode boot salah satu penyedia layanan aplikasi Anda:


Kalian bisa merender komponen yang ada didalam direktori komponen yang telah didaftarkan dengan cara : 

<x-flights::panel :flight="$flight" />

Building Layouts

Layouts Using Components

Penjelasan Layouts Using Components

Mendefinisikan komponen tata letak (layout) dengan satu komponen Blade. Misalnya, kita sedang membangun aplikasi daftar "todo" dengan tata letak seperti berikut:


Contoh Syntax 

<!-resources/views/components/layout.blade.php->
<html>
  <head>
<tittle>{{ $tittle ?? 'TodoManager'}}
   </tittle>
    </head>
   <body>
   <h1> Todos </h1>
 </hr>
{{ $slot }}
   </body>
</html>

Setelah komponen tata letak ditentukan,buatlah tampilan Blade yang menggunakan komponen tersebut. Dalam contoh ini, kita akan mendefinisikan tampilan sederhana yang menampilkan daftar tugas:


Contoh Syntax 

<!--resources/views/taks.blade.php-->
<x-layout>
     @foreach ($task as $task)
       {{$task}}
     @endforeach
</x-layout>

Mendefinisikan Layout

Mendefinisikan Layout

Dalam mendefinisikan layout, kita dapat menggunakan @section untuk memilih section tertentu dan @yield untuk menampilkan konten pada section yang dipilih.


Contoh Syntax

<html>
    <head>
        <>App Name - @yield('title')</>
    </head>
    <body>
        @section('sidebar')
            konten sidebar
        @show

        <div class="container">
            @yield('content')
        </div>
    </body>
</html>
Menampilkan konten pada section "sidebar"

Meng-extends Layout

Cara Meng-extends Layout

Setelah kita telah mendefinisikan layout, selanjutnya kita dapat mendefinisikan halaman "anak" (child page) yang akan mewarisi layout yang telah ditentukan.
Untuk melakukannya, kita harus menggunakan @extends untuk menentukan layout mana yang akan diwarisi. Halaman yang meng-inherit layout akan menampilkan konten pada @section dan akan ditampilkan menggunakan @yield.
@yield juga dapat menerima nilai default sebagai parameter kedua jika section tidak terdefinisi.


Contoh Syntax

@extends('layouts.app')

@section('title', 'Page Title')

@section('sidebar')
    @parent
<p>paragraf ini akan ditambahkan ke section "sidebar"</p>
@endsection

@section('content')
<p>isi konten pada section "content"</p>
@endsection


Dan

@yield('content', 'Default content')

CSRF Field

Penjelasan CSRF Field

Saat kita mendefinisikan form HTML, kita harus menambahkan CSRF (Cross-Site Request Forgery) untuk menjadi middleware dan memastikan keamanan dari request yang dikirim dengan menggunakan <@csrf>.


Contoh Syntax

<form method="POST" action="/profile">
    @csrf
    (isi form)
</form>
Blade template akan memvalidasi request sebelum diproses
