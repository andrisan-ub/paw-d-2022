Introduction

Introduction

Blade adalah fitur yang disediakan Laravel untuk proses templating sederhana namun sangat bermanfaat dalam proses pengembangan tampilan halaman web. Tidak seperti fitur templating PHP populer lainnya, Blade tidak membatasi pemrogram untuk menggunakan kode PHP biasa dalam membuat kode untuk tampilan.

Semua tampilan Blade dikompilasi ke dalam kode PHP biasa dan kemudian disimpan dalam cache hingga diubah, yang berarti Blade pada dasarnya tidak menambahkan beban atau overhead pada saat aplikasi dijalankan. Blade pada laravel menggunakan ekstensi file .blade.php dan biasanya disimpan di direktori resources / views. Dalam hal ini Blade pada Laravel menggunakan basis template inheritance dan sections.

Salah satu kegunaan dari Blade Template Engine Laravel adalah penggunaan layout, agar tampilan yang berulang misalnya header, footer, sidebar dan sebagainya tidak perlu dibuat berulang kali sehingga rawan inkonsistensi. Pada umumnya layout ditempatkan dalam sebuah folder yang bernama layout dalam folder views, namun demikian developer tidak dibatasi apabila ingin menggunakan nama lain.
Data dapat diteruskan ke tampilan Blade menggunakan “view helper’s second argument”: 

Route::get('/', function () {
                  return view('greeting', ['name' => 'Finn']);
               });
Displaying Data

HTML Entity Encoding

Secara umum, Blade akan menggandakan encode entitas HTML. Jika ingin menonaktifkan penyandian ganda, panggil method “Blade::withoutDoubleEncoding” dari method “boot” pada class 

<?php
 
                namespace App\Providers;
                 
                use Illuminate\Support\Facades\Blade;
                use Illuminate\Support\ServiceProvider;
                 
                class AppServiceProvider extends ServiceProvider
                {
                   /**
                    * Bootstrap any application services.
                    *
                    * @return void
                    */
                   public function boot()
                   {
                       Blade::withoutDoubleEncoding();
                   }
                }

Blade & JavaScript Frameworks

Karena banyak framework JavaScript juga menggunakan kurung kurawal untuk menampilkan ekspresi yang diberikan harus ditampilkan pada web browser, kita dapat menggunakan simbol @ untuk memberi tahu mesin rendering Blade bahwa ekspresi harus tetap tidak tersentuh. Contoh: 

 <h1>Laravel</h1>
 
                Hello, @{{ name }}.

Dalam contoh di atas, simbol @ akan dihapus oleh Blade. Namun, ekspresi {{ name }} akan tetap tidak tersentuh oleh mesin Blade, memungkinkan dirender oleh framework JavaScript.

Simbol @ juga dapat digunakan untuk menghindari pengarahan oleh Blade: 

{{-- Blade template --}}
                @@if()
                 
                <!-- HTML output -->
                @if()
Blade Directives

If Statements

Kita dapat membuat pernyataan if menggunakan @if, @elseif, @else, dan @endif. Ini memiliki fungsi identik dengan PHP : 

@if (count($records) === 1)
                I have one record!
             @elseif (count($records) > 1)
                I have multiple records!
             @else
                I don't have any records!
             @endif

Blade juga menyediakan @unless directive: 

@unless (Auth::check())
                You are not signed in.
             @endunless

Selain itu, @isset dan @empty dapat digunakan sebagai pintasan yang nyaman untuk fungsi PHP: 

@isset($records)
                // $records is defined and is not null...
             @endisset
              
             @empty($records)
                // $records is "empty"...
             @endempty

Switch Statements

Switch statement dapat dibuat menggunakan @switch, @case, @break, @default dan @endswitch.Contoh: 

@switch($i)
                @case(1)
                    First case...
                    @break
              
                @case(2)
                    Second case...
                    @break
              
                @default
                    Default case...
             @endswitch

Loops

Selain pernyataan bersyarat, Blade menyediakan arahan sederhana untuk bekerja dengan struktur loop PHP. Masing-masing berfungsi identik dengan counterpart PHP. Contoh: 

@for ($i = 0; $i < 10; $i++)
                The current value is {{ $i }}
             @endfor
              
             @foreach ($users as $user)
                <p>This is user {{ $user->id }}</p>
             @endforeach
              
             @forelse ($users as $user)
                <li>{{ $user->name }}</li>
             @empty
                <p>No users</p>
             @endforelse
              
             @while (true)
                <p>I'm looping forever.</p>
             @endwhile

Catatan:
Saat melakukan iterasi melalui loop foreach, kita dapat menggunakan variabel loop untuk mendapatkan informasi berharga tentang loop, seperti apakah kita berada di iterasi pertama atau terakhir melalui loop.

Saat menggunakan loop, kita juga dapat melewati iterasi saat ini atau mengakhiri loop menggunakan @continue dan @break: 

 @if ($user->type == 1)
                @continue
            @endif
          
            <li>{{ $user->name }}</li>
          
            @if ($user->number == 5)
                @break
            @endif
         @endforeach

Kita juga dapat menyertakan kondisi kelanjutan atau pemutusan. Contoh: 

@foreach ($users as $user)
                @continue($user->type == 1)
              
                <li>{{ $user->name }}</li>
              
                @break($user->number == 5)
             @endforeach

The Loop Variable

Saat melakukan perulangan melalui “loop foreach”, variabel $loop akan tersedia di dalam loop tersebut. Variabel ini akan menyediakan akses ke beberapa bit informasi yang berguna seperti indeks loop saat ini dan apakah saat ini literasi pertama atau terakhir pada loop: 

@foreach ($users as $user)
                  @if ($loop->first)
                      This is the first iteration.
                  @endif
                  @if ($loop->last)
                      This is the last iteration.
                  @endif
                  <p>This is user {{ $user->id }}</p>
               @endforeach

The Loop Variable-2

Dan jika Anda berada di dalam loop bersarang, Anda dapat mengakses variabel $loop loop induk melalui properti inangnya: 

@foreach ($users as $user)
            @foreach ($user->posts as $post)
              @if ($loop->parent->first)
                This is the first iteration of the parent loop.
              @endif
            @endforeach

The Loop Variable-3

Variabel $loop juga berisikan berbagai properti berguna lainnya:

Property Description
$loop->index Index iterasi loop saat ini (mulai dari 0).
$loop->iteration Iterasi loop saat ini (dimulai dari 1).
$loop->remaining Iterasi yang tersisa di dalam loop.
$loop->count Jumlah total item di dalam array yang diiterasi.
$loop->first Apakah ini iterasi pertama melalui loop.
$loop->last Apakah ini iterasi terakhir melalui loop.
$loop->even Apakah ini iterasi genap melalui loop.
$loop->odd Apakah ini iterasi ganjil melalui loop.
$loop->depth Tingkat bersarang dari loop saat ini.
$loop->parent Ketika dalam loop bersarang, variabel loop induk.

Conditional Classes

Perintah dari @class secara kondisional mengkompilasi String kelas CSS. Perintah menerima array kelas di mana kunci array berisi kelas atau kelas yang akan ditambahkan, sedangkan nilainy merupakan ekspresi boolean. Jika elemen array memilik kunci numerik tersebut, maka itu akan selalu dimasukkan ke dalam daftar kelas yang dirender: 


                  @php
                  $isActive = false;
                  $hasError = true;
               @endphp
                
               <span @class([
                  'p-4',
                  'font-bold' => $isActive,
                  'text-gray-500' => ! $isActive,
                  'bg-red' => $hasError,
               ])></span>
                
               <span class="p-4 text-gray-500 bg-red"></span>

Additional Attributes-2

Selain itu, perintah @disabled dapat digunakan untuk menujukkan apakah elemen yang diberikan harus di “disabled”: 

<button type="submit" @disabled($errors->isNotEmpty())>Submit</button>

Selain itu, perintah @readonly dapat digunakan untuk menunjukkan apakah elemen yang diberikan harus di “readonly”: 

<input type="email"
        name="email"
        value="email@laravel.com"
        @readonly($user->isNotAdmin())/>

Additional Attributes-3

Selain itu, perintah @required dapat digunakan untuk menunjukkan apakah elemen yang diberikan harus di “required”: 

<input type="text"
        name="title"
        value="title"
        @required($user->isAdmin())/>

Including Subviews

Meskipun Anda bebas menggunakan perintah @include, komponen Blade menyediakan fungsionalitas serupa dan menawarkan beberapa manfaat dibandingkan perintah @include seperti pengikatan data dan atribut.

Perintah @include Blade memungkinkan Anda untuk menyertakan tampilan Blade dari dalam tampilan lain. Semua variabel yang tersedia untuk tampilan induk akan tersedia untuk tampilan yang disertakan: 

<div>
   @include('shared.errors')
 
   <form>
       <!-- Form Contents -->
   </form>
</div>

Meskipun tampilan yang diikutsertakan akan mewarisi semua data yang tersedia dalam tampilan induk, Anda juga dapat meneruskan array data tambahan yang tersedia untuk tampilan yang disertakan: 

@include('view.name', ['status' => 'complete'])

Including Subviews-2

Jika Anda mencoba @include tampilan yang tidak ada, Laravel akan menampilkan error. Jika Anda ingin menyertakan tampilan yang mungkin ada atau mungkin tidak ada, Anda harus menggunakan perintah @includeIf: 

@includeIf('view.name', ['status' => 'complete'])

Jika Anda ingin @include tampilan jika ekspresi boolean yang diberikan dievaluasi menjadi TRUE or FALSE, Anda dapat menggunakan perintah @includeWhen dan @includeUnless: 

@includeWhen($boolean, 'view.name', ['status' => 'complete'])
@includeUnless($boolean, 'view.name', ['status' => 'complete'])

Untuk menyertakan tampilan pertama yang ada di array tampilan tertentu, Anda dapat mengunakan perintah @includeFirst: 

@includeFirst(['custom.admin', 'admin'], ['status' => 'complete'])
Anda harus menghindari penggunaan konstanta __DIR__ dan __FILE__ dalam tampilan Blade karena mereka akan merujuk ke lokasi tampilan yang di-cache dan dikompilasi.

The @Once Directive

perintah @once memungkinkan Anda untuk menentukan sebagian dari template yang hanya akan dievaluasi sekali persiklus rendering. Hal ini mungkin berguna untuk mendorong bagian JavaScript tertentu ke header halaman menggunakan stacks. Contohnya, jika Anda merender komponen tertentu dalam satu loop, Anda mungkin hanya ingin mendorong JavaScript ke header saat pertama kali komponen dirender: 

@once
   @push('scripts')
       <script>
           // Your custom JavaScript...
       </script>
   @endpush
@endonce

Karena perintah @once sering digunakan bersama dengan perintah @push dan @prepend, perintah @pushOnce dan @prependOnce tersedia untuk kemudahan penggunaan: 

@pushOnce('scripts')
   <script>
       // Your custom JavaScript...
   </script>
@endPushOnce

Raw PHP

Di beberapa situasi, menyematkan kode PHP sangat berguna ke dalam tampilan kita. Kita dapat menggunakan direktif Blade @php untuk mengeksekusi blok PHP biasa di dalam template kita. Jika dalam single statement, statement dapat dimasukkan atau diikutkan didalam direktif @php. 

 @php
    $counter = 1;
    @endphp

    Single statement:
    @php($counter = 1)

Comments

Blade juga memungkinkan kita untuk menentukan komentar dalam tampilan. Namun, tidak seperti komentar HTML, komentar dalam Blade tidak disertakan dalam HTML yang dikembalikan oleh aplikasi kita. 

{{-- This comment will not be present in the rendered HTML --}}
Components

Rendering Components

Untuk menampilkan komponen, kita dapat menggunakan tag komponen Blade dalam salah satu template Blade Anda. Tag komponen blade dimulai dengan string (x-) diikuti dengan nama case dari kelas komponen. Jika kelas komponen terletak lebih dalam di dalam direktori app/View/Components, kita dapat menggunakan karakter (.) untuk menunjukkan direktori. Misalnya, jika kita mengasumsikan sebuah komponen berlokasi di app/View/Components/Inputs/Button.php, kita dapat merendernya seperti pada contoh. 

Umumnya ditulis:
        <x-alert/>
 
        <x-user-profile/>
      
      Contoh:
      <x-inputs.button/>

Passing Data to Components

Kita dapat meneruskan data ke komponen Blade menggunakan atribut HTML. Nilai primitif dapat diteruskan ke komponen menggunakan string atribut HTML sederhana. Variabel PHP harus diteruskan ke komponen melalui atribut yang menggunakan karakter (:) sebagai awalan atau prefix.

Kita harus mendefinisikan semua atribut data komponen dalam konstruktor kelasnya. Semua properti publik pada komponen akan secara otomatis tersedia untuk tampilan komponen. Tidak perlu meneruskan data ke tampilan dari metode render komponen. 

<x-alert type="error" :message="$message"/>
<?php
 
                  namespace App\View\Components;
                   
                  use Illuminate\View\Component;
                   
                  class Alert extends Component
                  {
                      /**
                       * The alert type.
                       *
                       * @var string
                       */
                      public $type;
                   
                      /**
                       * The alert message.
                       *
                       * @var string
                       */
                      public $message;
                   
                      /**
                       * Create the component instance.
                       *
                       * @param  string  $type
                       * @param  string  $message
                       * @return void
                       */
                      public function __construct($type, $message)
                      {
                          $this->type = $type;
                          $this->message = $message;
                      }
                   
                      /**
                       * Get the view / contents that represent the component.
                       *
                       * @return \Illuminate\View\View|\Closure|string
                       */
                      public function render()
                      {
                          return view('components.alert');
                      }
                  }

Component Attributes

Terkadang kita mungkin perlu menentukan atribut HTML tambahan, seperti kelas, yang bukan merupakan bagian dari data yang diperlukan agar komponen berfungsi. Biasanya, kita ingin meneruskan atribut tambahan ini ke root element template komponen. Misalnya, bayangkan kita ingin merender komponen peringatan seperti pada contoh.

Semua atribut yang bukan bagian dari konstruktor komponen akan secara otomatis ditambahkan ke komponen “attributes bag”. Komponen teersebut secara otomatis tersedia melalui variabel $attributes. Semua atribut dapat dirender dalam komponen dengan menggemakan variabel dibawah. 

Contoh: 
  <x-alert type="error" :message="$message" class="mt-4"/>
                
  Variabel untuk merender atribut:
    <div {{ $attributes }}>
    <!-- Component content -->
    </div>

Secara default, beberapa kata kunci dicadangkan untuk penggunaan internal Blade untuk merender komponen. Kata kunci berikut tidak dapat didefinisikan sebagai properti publik atau nama metode dalam komponen Anda:


            - Data

            - withName

Slots

Anda sering perlu meneruskan konten tambahan ke komponen Anda melalui "slot". Slot komponen dirender dengan menggemakan variabel $slot. Untuk menjelajahi konsep ini, mari kita bayangkan bahwa komponen peringatan memiliki markup berikut:

<div class="alert alert-danger">
                      {{ §slot }}
                  </div>

juga dapat meneruskan konten ke slot dengan memasukkan konten ke dalam komponen: 

  <x-alert>
    <strong>Whoops!</strong> Something went wrong!
</x-alert>

Terkadang komponen mungkin perlu merender beberapa slot berbeda di lokasi yang berbeda dalam komponen. Mari kita ubah komponen peringatan kita untuk memungkinkan injeksi slot "judul": 

<span class="alert-title">{{ $title }}</span>
 
<div class="alert alert-danger">
    {{ $slot }}
</div>

Anda dapat menentukan konten slot bernama menggunakan tag x-slot. Konten apa pun yang tidak berada dalam tag x-slot eksplisit akan diteruskan ke komponen dalam variabel $slot: 

<x-alert>
    <x-slot:title>
        Server Error
    </x-slot>
 
    <strong>Whoops!</strong> Something went wrong!
</x-alert>

#Scooped Slots

Jika Anda telah menggunakan kerangka kerja JavaScript seperti Vue, Anda mungkin akrab dengan "slot terbatas", yang memungkinkan Anda mengakses data atau metode dari komponen di dalam slot Anda. Anda dapat mencapai perilaku serupa di Laravel dengan mendefinisikan metode publik atau properti pada komponen Anda dan mengakses komponen dalam slot Anda melalui variabel $component. Dalam contoh ini, kita akan mengasumsikan bahwa komponen x-alert memiliki metode formatAlert publik yang ditentukan pada kelas komponennya: 

<x-alert>
    <x-slot:title>
        {{ $component->formatAlert('Server Error') }}
    </x-slot>
 
    <strong>Whoops!</strong> Something went wrong!
</x-alert>

#Slots Atribut

Seperti komponen Blade, Anda dapat menetapkan atribut tambahan ke slot seperti nama kelas CSS: 

 <x-card class="shadow-sm">
    <x-slot:heading class="font-bold">
        Heading
    </x-slot>
 
    Content
 
    <x-slot:footer class="text-sm">
        Footer
    </x-slot>
</x-card>

Inline Component Views

Untuk komponen yang sangat kecil, mungkin terasa rumit untuk mengelola kelas komponen dan template tampilan komponen. Untuk alasan ini, Anda dapat mengembalikan markup komponen langsung dari metode render: 

public function render()
{
    return <<<'blade'
        <div class="alert alert-danger">
            {{ $slot }}
        </div>
    blade;
}

Dynamic Components

Terkadang Anda mungkin perlu merender komponen tetapi tidak tahu komponen mana yang harus dirender hingga runtime. Dalam situasi ini, Anda dapat menggunakan komponen komponen dinamis bawaan Laravel untuk merender komponen berdasarkan nilai runtime atau variabel: 

<x-dynamic-component :component="$componentName" class="mt-4" />

Manually Registering Components

Saat menulis komponen untuk aplikasi Anda sendiri, komponen secara otomatis ditemukan dalam direktori app/View/Components dan direktori resources/views/components. Namun, jika Anda membangun paket yang menggunakan komponen Blade atau menempatkan komponen di direktori non-konvensional, Anda harus mendaftarkan kelas komponen Anda secara manual dan alias tag HTML-nya sehingga Laravel tahu di mana menemukan komponen tersebut. Anda biasanya harus mendaftarkan komponen Anda dalam metode boot penyedia layanan paket Anda:

use IlluminateSupportFacadesBlade;
use VendorPackageViewComponentsAlertComponent;
 
/**
 * Bootstrap your package's services.
 *
 * @return void
 */
public function boot()
{
    Blade::component('package-alert', AlertComponent::class);
}

Autoloading Package Components

Anda dapat menggunakan metode componentNamespace untuk memuat kelas komponen secara otomatis berdasarkan konvensi. Misalnya, paket Nightshade mungkin memiliki komponen Kalender dan ColorPicker yang berada dalam namespace PackageViewsComponents:

use IlluminateSupportFacadesBlade;
 
/**
 * Bootstrap your package's services.
 *
 * @return void
 */
public function boot()
{
    Blade::componentNamespace('Nightshade\Views\Components', 'nightshade');
}
Anonymous Components

Anonymous Index Components

Terkadang, ketika sebuah komponen terdiri dari banyak template Blade, Anda mungkin ingin mengelompokkan template komponen yang diberikan dalam satu direktori. Misalnya, bayangkan komponen "accordion" dengan struktur direktori berikut: 

/resources/views/components/accordion.blade.php
/resources/views/components/accordion/item.blade.php

Struktur direktori ini memungkinkan Anda untuk merender komponen “accordion” dan itemnya seperti ini: 

<x-accordion>
   <x-accordion.item>
       ...
   </x-accordion.item>
</x-accordion>

Anonymous Index Components-2

Namun, untuk merender komponen “accordion” melalui x-accordion, kami terpaksa menempatkan templat komponen accordion "index" di direktori resources/views/components alih-alih menumpuknya di dalam direktori accoridon dengan templat terkait accordion lainnya. Untungnya, Blade memungkinkan Anda untuk menempatkan file index.blade.php di dalam direktori template komponen. Ketika template index.blade.php ada untuk komponen, itu akan dirender sebagai simpul "root" dari komponen. Jadi, kita dapat terus menggunakan sintaks Blade yang sama seperti yang diberikan pada contoh di atas; namun, kami akan menyesuaikan struktur direktori kami seperti ini: 

/resources/views/components/accordion/index.blade.php
/resources/views/components/accordion/item.blade.php

Data Properties / Attributes

Meskipun komponen anonim tidak memiliki kelas yang terkait, anda tetap dapat menentukan atribut mana yang harus dianggap sebagai variabel dan atribut mana yang hharus ditempatkan di tas atribut komponen menggunakan arahan @props di bagian atas template Blade komponen Anda untuk membedakan data mana yag harus di teruskan ke komponen sebagai atirbut. Semua atribut lain pada komponen akan tersedia melalui tas atribut komponen. Jika Anda ingin memberikan nilai default pada variabel data, Anda dapat menentukan nama variabel sebagai kunci array dan nilai default sebagai nilai array: 

<!-- /resources/views/components/alert.blade.php -->
 
@props(['type' => 'info', 'message'])
 
<div {{ $attributes->merge(['class' => 'alert alert-'.$type]) }}>
   {{ $message }}
</div>

Berdasarkan pendefinisian komponen di atas, kita dapat membuat komponen seperti ini: 

<x-alert type="error" :message="$message" class="mb-4"/>

Accessing Parent Data

Untuk mengakses data dari komponen induk di dalam komponen turunan Anda dapat menggunakan arahan @aware. Misalnya, kita sedang membangun komponen menu kompleks yang terdiri dari induk <x-menu> dan anak <x-menu.item>, maka: 

<x-menu color="purple">
   <x-menu.item>...</x-menu.item>
   <x-menu.item>...</x-menu.item>
</x-menu>

Komponen <x-menu> dapat memiliki implementasi seperti berikut: 

<!-- /resources/views/components/menu/index.blade.php -->
 
@props(['color' => 'gray'])
 
<ul {{ $attributes->merge(['class' => 'bg-'.$color.'-200']) }}>
   {{ $slot }}
</ul>

Accessing Parent Data-2

Karena prop warna ('color') hanya diteruskan ke induk (<x-menu>), itu tidak akan tersedia di dalam <x-menu.item>. Namun, jika kita menggunakan direktif @aware, kita juga dapat membuatnya tersedia di dalam <x-menu.item>: 

<!-- /resources/views/components/menu/item.blade.php -->
 
@aware(['color' => 'gray'])
 
<li {{ $attributes->merge(['class' => 'text-'.$color.'-800']) }}>
   {{ $slot }}
</li>

@aware tidak dapat mengakses data induk yang tidak secara eksplisit diteruskan ke komponen induk melalui atribut HTML. Nilai @props default yang tidak secara eksplisit diteruskan ke komponen induk tidak dapat diakses oleh direktif @aware.

Anonymous Components Namespaces

Komponen anonim biasanya ditentukan dengan menempatkan template Blade di dalam direktori resource/views/components Anda. Namun, terkadang Anda mungkin ingin mendaftarkan jalur komponen anonim lainnya dengan Laravel selain jalur default. Untuk memberi tahu Laravel tentang lokasi komponen anonim ini, Anda dapat menggunakan metode anonymousComponentNamespace yang disediakan oleh Blade.

Metode anonymousComponentNamespace menerima "jalur" ke lokasi komponen anonim sebagai argumen pertama dan "namespace" yang komponennya harus ditempatkan sebagai argumen kedua. Seperti yang akan Anda lihat pada contoh di bawah, "namespace" akan diawali dengan nama komponen saat komponen dirender. Biasanya, metode ini harus dipanggil dari metode boot salah satu penyedia layanan aplikasi Anda: 

/**
* Bootstrap any application services.
*
* @return void
*/
public function boot()
{
   Blade::anonymousComponentNamespace('flights.bookings.components', 'flights');
}

Anda dapat merender komponen panel yang ada di dalam direktori komponen yang baru didaftarkan seperti:

<x-flights::panel :flight="$flight" />
Building Layouts

Layouts Using Components

Mendefinisikan komponen tata letak (layout) dengan satu komponen Blade. Misalnya, kita sedang membangun aplikasi daftar "todo" dengan tata letak seperti berikut: 

<!-- resources/views/components/layout.blade.php -->
 
<html>
   <head>
       <title>{{ $title ?? 'Todo Manager' }}</title>
   </head>
   <body>
       <h1>Todos</h1>
       <hr/>
       {{ $slot }}
   </body>
</html>

Setelah komponen tata letak ditentukan,buatlah tampilan Blade yang menggunakan komponen tersebut. Dalam contoh ini, kita akan mendefinisikan tampilan sederhana yang menampilkan daftar tugas kita: 

<!-- resources/views/tasks.blade.php -->
 
<x-layout>
   @foreach ($tasks as $task)
       {{ $task }}
   @endforeach
</x-layout>

Layouts Using Components-2

Konten yang dimasukkan ke dalam komponen akan diberikan ke variabel $slot default di dalam komponen layout kita. Tata letak di atasjuga mengikuti slot $title jika tersedia; jika tidak, judul default akan ditampilkan. Kami dapat memasukkan judul khusus dari tampilan daftar tugas kami menggunakan sintaks slot standar yang dibahas dalam dokumentasi komponen: 

<!-- resources/views/tasks.blade.php -->
 
<x-layout>
   <x-slot:title>
       Custom Title
   </x-slot>
 
   @foreach ($tasks as $task)
       {{ $task }}
   @endforeach
</x-layout>

Lalu, kita tinggal mengembalikan tampilan “task” dari sebuah “route”: 

use AppModelsTask;
 
Route::get('/tasks', function () {
   return view('tasks', ['tasks' => Task::all()]);
});

Layouts Using Template Inheritance

Layout juga dapat dibuat melalui "Template Inheritence". Ini adalah cara utama membangun aplikasi sebelum pengenalan components. Untuk memulai, mari kita lihat contoh sederhana. Pertama, kita akan memeriksa layout halaman. Karena sebagian besar aplikasi web mempertahankan layout umum yang sama di berbagai halaman, akan lebih mudah untuk mendefinisikan layout sebagai tampilan Single Blade. 

<!-- resources/views/layouts/app.blade.php -->
 
                  <html>
                      <head>
                          <title>App Name - @yield('title')</title>
                      </head>
                      <body>
                          @section('sidebar')
                              This is the master sidebar.
                          @show
                   
                          <div class="container">
                              @yield('content')
                          </div>
                      </body>
                  </html>
Forms

CSRF Field

Setiap kali kita mendefinisikan formulir HTML di aplikasi, kita harus menyertakan CSRF token field yang tersembunyi di formulir sehingga CSRF protection middleware dapat memvalidasi permintaan. Kita juga dapat menggunakan directive Blade @csrf untuk menghasilkan token field. 

<form method="POST" action="/profile">
              @csrf
               
                  ...
              </form>

Method Field

Karena formulir HTML tidak dapat membuat permintaan PUT, PATCH, atau DELETE, kita perlu menambahkan _method yang tersembunyi untuk menyamarkan kata kerja pada HTTP. Arahan direktif Blade @method dapat membuat field ini untuk kita. 

<form action="/foo/bar" method="POST">
                  @method('PUT')
               
                  ...
              </form>

Validation Errors

Direktif @error dapat digunakan untuk memeriksa dengan cepat apakah ada pesan kesalahan validasi untuk atribut tertentu. Dalam direktif @error, kita dapat menggunakan variabel $message untuk menampilkan pesan kesalahan. 

!-- /resources/views/post/create.blade.php -->
 
                  <label for="title">Post Title</label>
                   
                  <input id="title"
                      type="text"
                      class="@error('title') is-invalid @enderror">
                   
                  @error('title')
                      <div class="alert alert-danger">{{ $message }}</div>
                  @enderror
