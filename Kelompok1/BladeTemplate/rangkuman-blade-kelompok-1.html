Rangkuman Blade Template

Introduction

Blade merupakan template engine bawaan dari laravel. Blade menawarkan syntax yang lebih mudah dan singkat untuk dipakai dalam menghasilkan dokumen HTML. Tidak seperti templating engine PHP lainnya, Blade tidak membatasi kita untuk menggunakan kode PHP biasa dalam penulisan template. File blade template menggunakan ekstensi .blade.php dan biasanya disimpan di direktori resources/views. Tampilan blade dapat dikembalikan dari route atau controller menggunakan global view helper. Tentu saja, seperti yang disebutkan dalam dokumentasi tentang views, data dapat diteruskan ke tampilan Blade menggunakan argumen kedua view helper : 

   Route::get('/', function () {
       return view('greeting', ['name' => 'Finn']);
   });

Displaying Data

Kita dapat menampilkan data yang diteruskan ke tampilan Blade Anda dengan membungkus variabel dalam kurung kurawal. Sebagai contoh, diberikan route berikut: 

   Route::get('/', function () {
     return view('welcome', ['name' => 'Samantha']);
   });

Kita dapat menampilkan isi dari variabel name dengan : 

Hello, {{ $name }}.

HTML Entity Encoding

Secara default, Blade (dan Laravel e helper) akan menggandakan encode entitas HTML. Jika kita ingin menonaktifkan double encoding, panggil method Blade::withoutDoubleEncoding dari boot method AppServiceProvider kita : 

<?php
namespace App\Providers;
use Illuminate\Support\Facades\Blade;
use Illuminate\Support\ServiceProvider;
class AppServiceProvider extends ServiceProvider {
 /**
   * Bootstrap any application services.
   *
   * @return void
   */
 public function boot(){
     Blade::withoutDoubleEncoding();
 }
}

# Displaying Unescaped Data

Secara default, statement Blade {{ }} secara otomatis dikirim melalui function htmlspecialchars PHP untuk mencegah serangan XSS. Jika Anda tidak ingin data kita di-escape, kita dapat menggunakan syntax berikut: 

Hello, {!! $name !!}.

Blade & JavaScript Frameworks

Karena banyak framework JavaScript juga menggunakan kurung kurawal untuk menunjukkan ekspresi yang diberikan harus ditampilkan di browser, kita dapat menggunakan simbol @ untuk memberi tahu Blade rendering machine bahwa ekspresi harus tetap tidak tersentuh. Sebagai contoh: 

<h1>Laravel</h1>
Hello, @{{ name }}.

Dalam contoh ini, simbol @ akan dihapus oleh Blade; namun, ekspresi {{ name }} akan tetap tidak tersentuh oleh mesin Blade, memungkinkannya untuk di render oleh framework JavaScript kita.

# Rendering JSON

Terkadang kita dapat meneruskan sebuah array ke tampilan kita dengan maksud untuk merendernya sebagai JSON untuk menginisialisasi variabel JavaScript. Sebagai contoh: 

<script>
   var app = <?php echo json_encode($array); ?>;
</script>

Namun, alih-alih memanggil json_encode secara manual, kita dapat menggunakan arahan method Illuminate\Support\Js::from. Method from menerima argumen yang sama dengan fungsi json_encode PHP; namun, ini akan memastikan bahwa JSON yang dihasilkan diloloskan dengan benar untuk dimasukkan dalam tanda kutip HTML. Metode from akan mengembalikan pernyataan JavaScript string JSON.parse yang akan mengubah object atau array yang diberikan menjadi object JavaScript yang valid: 

<script>
   var app = {{ Illuminate\Support\Js::from($array) }};
</script>

# The @verbatim Directive

Jika kita ingin menampilkan variabel JavaScript di sebagian besar template kita, kita dapat membungkus HTML di directive @verbatim sehingga kita tidak perlu mengawali setiap pernyataan echo Blade dengan simbol @ : 

@verbatim
   <div class="container">
       Hello, {{ name }}.
   </div>
@endverbatim

Blade Directives

Selain pewarisan template dan tampilan data, Blade juga menyediakan shortcut yang nyaman untuk struktur kontrol PHP umum, seperti conditional statements dan loop. Shortcut ini menyediakan cara yang sangat bersih dan singkat untuk bekerja dengan struktur kontrol PHP sementara juga tetap familiar dengan PHP counterpart mereka.

If Statements

Kita dapat membuat pernyataan if menggunakan directive @if, @elseif, @else dan @endif. Arahan ini berfungsi identik dengan PHP counterparts mereka : 

@if (count($records) === 1)
   I have one record!
@elseif (count($records) > 1)
   I have multiple records!
@else
   I don't have any records!
@endif

Blade juga menyediakan @unless directive : 

@unless (Auth::check())
   You are not signed in.
@endunless

# Authentication Directives

@auth dan @guest dapat digunakan untuk menentukan dengan cepat apakah pengguna saat ini diautentikasi atau merupakan guest : 

@auth
   // The user is authenticated...
@endauth
@guest
   // The user is not authenticated...
@endguest

# Environment Directives

Kita dapat memeriksa apakah aplikasi berjalan di production environment menggunakan arahan @production : 

@production
   // Production specific content...
@endproduction

Atau, kita dapat menentukan apakah aplikasi berjalan di environment tertentu menggunakan directive @env : 

@env('staging')
   // The application is running in "staging"...
@endenv
@env(['staging', 'production'])
   // The application is running in "staging" or "production"...
@endenv

# Section Directives

Kita dapat menentukan apakah template inheritance section memiliki konten menggunakan directive @hasSection : 

@hasSection('navigation')
   <div class="pull-right">
       @yield('navigation')
   </div>
   <div class="clearfix"></div>
@endif

Kita dapat menggunakan directive sectionMissing untuk menentukan apakah suatu bagian tidak memiliki konten : 

@sectionMissing('navigation')
   <div class="pull-right"></div>
       @include('default-navigation')
   </div>
@endif

Switch

Pernyataan switch dapat dibangun menggunakan direktif @switch, @case, @break, @default dan @endswitch 

   @switch($i)
       @case(1)
           First case...
           @break

       @case(2)
           Second case...
           @break

       @default
           Default case...
   @endswitch

Loops

Selain pernyataan bersyarat, Blade menyediakan arahan sederhana untuk bekerja dengan struktur loop PHP. Sekali lagi, masing-masing arahan ini berfungsi identik dengan rekan PHP mereka: 

   @for($i = 0; $i < 10; $i++)
       The current value is {{ $i }}
   @endfor

   @foreach ($users as $user)
       <p>This is user {{ $user->id }}</p>
   @endforeach

   @forelse ($users as $user)
       <li>{{ $user->name }}</li>
   @empty
       <p>No users</p>
   @endforelse

   @while (true)
       <p>I'm looping forever.</p>
   @endwhile

Fun Fact

Saat melakukan iterasi melalui foreach loop, kita dapat menggunakan variabel loop untuk mendapatkan informasi berharga tentang loop, seperti apakah kita berada di iterasi pertama atau terakhir melalui loop.

Saat menggunakan loop, kita juga dapat melewati iterasi saat ini atau mengakhiri loop menggunakan arahan @continue dan @break 

   @foreach ($users as $user)
       @if ($user->type == 1)
           @continue
       @endif

       <li>{{ $user->name }}</li>

       @if ($user->number == 5)
           @break
       @endif
   @endforeach

Kita juga dapat menyertakan kondisi kelanjutan atau pemutusan dalam deklarasi direktif. 

   @foreach ($users as $user)
       @continue($user->type == 1)

       <li>{{ $user->name }}</li>

       @break($user->number == 5)
   @endforeach

The Loop Variable

Saat melakukan iterasi melalui sebuah foreach loop, sebuah $loop variabel akan tersedia di dalam loop. Variabel ini menyediakan akses ke beberapa bit informasi yang berguna seperti indeks loop saat ini dan apakah ini adalah iterasi pertama atau terakhir melalui loop: 

   @foreach ($users as $user)
       @if ($loop->first)
           This is the first iteration.
       @endif

       @if ($loop->last)
           This is the last iteration.
       @endif

       <p>This is user {{ $user->id }}</p>
   @endforeach

Jika kita berada dalam loop bersarang, kita dapat mengakses $loop variable loop induk melalui parent properti: 

   @foreach ($users as $user)
       @foreach ($user->posts as $post)
           @if ($loop->parent->first)
               This is the first iteration of the parent loop.
           @endif
       @endforeach
   @endforeach

Kita juga dapat menyertakan kondisi kelanjutan atau pemutusan dalam deklarasi direktif. 

   @foreach ($users as $user)
       @continue($user->type == 1)

       <li>{{ $user->name }}</li>

       @break($user->number == 5)
   @endforeach

Variabel $loop juga berisi berbagai properti berguna lainnya:

Property Description
$loop->index Indeks dari iterasi loop saat ini (dimulai dari 0).
$loop->iteration Iterasi loop saat ini (dimulai dari 1).
$loop->remaining Iterasi yang tersisa dalam loop.
$loop->count Jumlah total item dalam array yang diulang.
$loop->first Apakah ini adalah iterasi pertama melalui loop.
$loop->last Apakah ini adalah iterasi terakhir melalui loop.
$loop->even Apakah ini merupakan iterasi genap melalui loop.
$loop->odd Apakah ini iterasi ganjil melalui loop.
$loop->depth Tingkat bersarang dari loop saat ini.
$loop->parent Saat berada dalam loop bersarang, variabel loop induk.

Conditional Classes

Direktif @class mengkompilasi string kelas CSS secara kondisional. Arahan menerima array kelas di mana kunci array berisi kelas atau kelas yang ingin Anda tambahkan, sedangkan nilainya adalah ekspresi boolean. Jika elemen array memiliki kunci numerik, elemen tersebut akan selalu disertakan dalam daftar kelas yang dirender: 

@php
    $isActive = false;
    $hasError = true;
@endphp

<span @class([
    'p-4',
    'font-bold' => $isActive,
    'text-gray-500' => ! $isActive,
    'bg-red' => $hasError,
])></span>

<span class="p-4 text-gray-500 bg-red"></span>

Additional Attributes

Untuk kenyamanan, kita dapat menggunakan @checked direktif untuk dengan mudah menunjukkan jika input kotak centang HTML yang diberikan "dicentang". Arahan ini akan bergema checked jika kondisi yang diberikan mengevaluasi ke true: 

<input type="checkbox"
       name="active"
       value="active"
       @checked(old('active', $user->active)) />

Demikian juga, @selected arahan dapat digunakan untuk menunjukkan apakah opsi pilih yang diberikan harus "dipilih": 

<select name="version">
    @foreach ($product->versions as $version)
        <option value="{{ $version }}" @selected(old('version') == $version)>
           {{ $version }}
        </option>
    @endforeach
</select>

Selain itu, @disabled arahan dapat digunakan untuk menunjukkan apakah elemen yang diberikan harus "dinonaktifkan": 

<button type="submit" @disabled($errors->isNotEmpty())>Submit</button>

Selain itu, @readonly arahan dapat digunakan untuk menunjukkan apakah elemen yang diberikan harus "hanya-baca": 

<input type="email"
       name="email"
       value="email@laravel.com"
       @readonly($user->isNotAdmin()) />

Selain itu, $required arahan dapat digunakan untuk menunjukkan apakah elemen yang diberikan harus "diperlukan": 

<input type="text"
       name="title"
       value="title"
       @required($user->isAdmin()) />

Including Subviews

Fun Fact

Meskipun kita bebas menggunakan direktif, komponen @include Blade menyediakan fungsionalitas serupa dan menawarkan beberapa manfaat dibandingkan direktif seperti pengikatan data dan atribut.

Arahan Blade @include memungkinkan kita untuk menyertakan tampilan Blade dari dalam tampilan lain. Semua variabel yang tersedia untuk tampilan induk akan tersedia untuk tampilan yang disertakan: 

<div>
    @include('shared.errors')

    <form>
        <!-- Form Contents -->
    </form>
</div>

Meskipun tampilan yang disertakan akan mewarisi semua data yang tersedia di tampilan induk, kita juga dapat meneruskan larik data tambahan yang harus tersedia untuk tampilan yang disertakan: 

@include('view.name', ['status' => 'complete'])

Jika kita mencoba untuk @include melihat yang tidak ada, Laravel akan membuat kesalahan. Jika kita ingin menyertakan tampilan yang mungkin ada atau tidak, kita harus menggunakan @includeIf arahan: 

@includeIf('view.name', ['status' => 'complete'])

Jika Anda ingin @include melihat apakah ekspresi boolean yang diberikan dievaluasi ke true or false, Anda dapat menggunakan arahan @includeWhen dan @includeUnless 

@includeWhen($boolean, 'view.name', ['status' => 'complete'])

@includeUnless($boolean, 'view.name', ['status' => 'complete'])

Untuk menyertakan tampilan pertama yang ada dari larik tampilan tertentu, Anda dapat menggunakan includeFirst direktif: 

@includeFirst(['custom.admin', 'admin'], ['status' => 'complete'])

But be Careful!

Kita harus menghindari penggunaan __DIR__ dan __FILE__ konstanta dalam tampilan Blade, karena mereka akan merujuk ke lokasi tampilan yang di-cache dan dikompilasi.

Rendering Views for Collection

Kita dapat menggabungkan loop dan memasukkan ke dalam satu baris dengan @each arahan Blade: 

@each('view.name', $jobs, 'job')

Argumen @each pertama direktif adalah tampilan untuk di render untuk setiap elemen dalam array atau koleksi. Argumen kedua adalah larik atau koleksi yang ingin diulangi, sedangkan argumen ketiga adalah nama variabel yang akan ditetapkan ke iterasi saat ini dalam tampilan. Jadi, misalnya, jika kita mengulangi array jobs, biasanya kita ingin mengakses setiap pekerjaan sebagai job variabel dalam tampilan. Kunci array untuk iterasi saat ini akan tersedia sebagai key variabel dalam tampilan.

Kita juga dapat memberikan argumen keempat ke @each direktif. Argumen ini menentukan tampilan yang akan di render jika array yang diberikan kosong. 

@each('view.name', $jobs, 'job', 'view.empty')

But be Careful!

Tampilan yang dirender melalui @each tidak mewarisi variabel dari tampilan induk. Jika tampilan anak memerlukan variabel-variabel ini, kita harus menggunakan arahan @foreach dan sebagai gantinya.

The @once Directive

@once Directive memungkinkan kita untuk menentukan bagian dari template yang hanya akan dievaluasi sekali per siklus rendering. Ini mungkin berguna untuk menjalankan dahulu bagian tertentu dari JavaScript ke dalam header halaman menggunakan stacks. Seperti contoh, jika kita merender komponen tertentu ke dalam suatu perulangan, kita mungkin hanya ingin mendorong JavaScript ke header saat pertama kali komponen dirender : 

   @once
       @push('scripts')
           <script>
               // Your custom JavaScript...
           </script>
       @endpush
   @endonce

Karena @once Directive sering digunakan bersama dengan direktif @push atau @prepend, direktif @pushOnce dan @prependOnce tersedia untuk kenyamanan Anda: 

   @pushOnce('scripts')
       <script>
           // Your custom JavaScript...
       </script>
   @endPushOnce

Raw PHP

Raw PHP sangat berguna untuk menyematkan kode PHP ke dalam tampilan Anda. Kita dapat menggunakan direktif Blade @php untuk mengeksekusi blok PHP biasa di dalam template yang kita buat. Contoh penggunaan Raw PHP : 

   @php
       $counter = 1;
   @endphp

Kita hanya perlu menulis satu pernyataan PHP untuk dapat memanggil statement yang telah kita buat di Raw PHP, seperti contoh code yang ada di bawah :

   @php($counter = 1)

Comment

Blade juga memungkinkan kita untuk melakukan komentar di dalam code. Komentar di blade juga tidak akan ada dalam html yang akan ditampilkan. Contoh penggunaan code yang akan dikomen :

   {{-- This comment will not be present in the rendered HTML --}}

Components

Components dan slots memberikan manfaat yang serupa dengan sections, layouts, and includes. Ada dua pendekatan untuk menulis komponen: komponen berbasis kelas dan komponen anonim. Untuk membuat komponen berbasis kelas, Kita dapat menggunakan perintah make:component. Perintah make:component akan menempatkan komponen di direktori app/View/Components. Tambahkan perintah dibawah di terminal untuk membuat component : 

   php artisan make:component “nama_file”

Perintah make:component juga akan membuat template tampilan untuk komponen. Tampilan akan ditempatkan di direktori resources/views/components. Saat menulis komponen untuk aplikasi Anda sendiri, komponen secara otomatis ditemukan dalam direktori app/View/Components dan direktori resources/views/components, jadi biasanya tidak diperlukan registrasi komponen lebih lanjut.

Anda juga dapat membuat komponen di dalam subdirektori dengan perintah dibawah :

   php artisan make:component “nama_folder_subDirektori”/“nama_file”

Perintah di atas akan membuat komponen "nama_file" di direktori app/View/Components/“nama_folder_subDirektori” dan view akan ditempatkan di direktori resources/views/components/“nama_folder_subDirektori.

Jika Anda ingin membuat komponen anonim (komponen dengan hanya menambahkan template Blade di resources dan tanpa menambahkan kelas di direktori app/View/Components), Anda dapat menggunakan flag --view saat menjalankan perintah make:component seperti contoh perintah dibawah : 

   php artisan make:component “nama_folder”.“nama_file” --view

Perintah di atas akan membuat file Blade di direktori resources/views/components/”nama_folder”/”nama_file”.blade.php yang dapat dijalankan sebagai komponen dengan perintah <x-”nama_folder”.”nama_file” />.

Rendering Components

Untuk menampilkan komponen, Kita dapat menggunakan tag komponen Blade dalam salah satu template Blade kita. Tag komponen blade dimulai dengan string x- diikuti dengan nama class dari kelas komponen. Contoh perintah : 

   <x-header/>

Jika kelas komponen berada dalam folder lebih dalam di dalam direktori app/View/Components, Kita dapat menggunakan . karakter untuk menunjukkan direktori dalam forder tersebut. Misalnya, jika kita mengasumsikan sebuah komponen terletak di app/View/Components/Inputs/Button.php, kita dapat menjalankannya seperti ini: 

   <x-inputs.button/>

Passing Data To Components

Anda dapat meneruskan data ke komponen Blade menggunakan atribut HTML. Hard-coded, nilai primitif dapat diteruskan ke komponen menggunakan string atribut HTML sederhana. Ekspresi dan variabel PHP harus diteruskan ke komponen melalui atribut yang menggunakan karakter : sebagai awalan. Contoh code : 

   <x-alert type="error" :message="$message"/>

Anda harus mendefinisikan semua atribut data komponen dalam konstruktor kelasnya. Semua properti publik pada komponen akan secara otomatis tersedia untuk tampilan komponen. Tidak perlu meneruskan data ke tampilan dari metode render komponen:

<?php namespace App\View\Components; use Illuminate\View\Component; class Alert extends Component { /** * The alert type. * * @var string */ public $type; /** * The alert message. * * @var string */ public $message; /** * Create the component instance. * * @param string $type * @param string $message * @return void */ public function __construct($type, $message) { $this->type = $type; $this->message = $message; } /** * Get the view / contents that represent the component. * * @return \Illuminate\View\View|\Closure|string */ public function render() { return view('components.alert'); } }

Saat komponen Anda dirender, Anda dapat menampilkan konten variabel publik komponen Anda dengan menggunakan variabel berdasarkan nama. Contoh code :

   <div class="alert alert-{{ $type }}">
       {{ $message }}
   </div>

Manually Registering Components

Jika kita tidak menulis sebuah package, bagian dari dokumentasi ini tidak relevan bagi kita. Komponen ini berlaku untuk mereka yang menulis packages yang menyertakan view components.

Jika kamu membuat sebuah package yang menggunakan komponen blade atau menempatkan komponen dalam direktori non-konvensional, Kita perlu mendaftarkan kelas komponen dan tag HTML-nya secara manual sehingga Laravel tahu di mana menemukan komponen tersebut. Kita biasanya harus mendaftarkan komponen dalam method boot dari penyedia layanan paket kita

Setelah komponen Anda terdaftar, komponen tersebut dapat di render menggunakan alias tag-nya:

<x-package-alert/>


Autoloading Package Components

Anda dapat menggunakan metode componentNamespace untuk memuat kelas komponen secara otomatis berdasarkan konvensi. Blade akan secara otomatis mendeteksi kelas yang ditautkan ke komponen ini dengan pascal-casing nama komponen. Subdirektori juga didukung menggunakan notasi "titik".

Anonymous Components

komponen anonim menyediakan mekanisme untuk mengelola komponen melalui satu file. Untuk menentukan komponen anonim, hanya perlu menempatkan template Blade di dalam direktori resources/views/components kita. Misalnya, dengan asumsi kita telah mendefinisikan sebuah komponen di resources/views/components/alert.blade.php, kita cukup merendernya seperti ini: 

   <x-alert/>

Kita dapat menggunakan (.) untuk menunjukkan jika komponen bersarang lebih dalam di dalam direktori components, resources/views/components/inputs/button.blade.php, kita dapat merendernya seperti ini: 

   <x-inputs.button/>

ketika sebuah komponen terdiri dari banyak template Blade, kita mungkin ingin mengelompokkan template komponen yang diberikan dalam satu direktori.

/resources/views/components/accordion/item.blade.php

Struktur direktori ini memungkinkan untuk merender komponen akordeon dan itemnya seperti: 

   <x-accordion>
     <x-accordion.item>
         ...
     </x-accordion.item>
   </x-accordion>

Untuk merender komponen akordeon melalui x-accordion, kita harus menempatkan templat komponen accordion "indeks" di resources/views/components alih-alih menumpuknya di dalam direktori accordion dengan templat terkait accordion lainnya. 

/resources/views/components/accordion/index.blade.php
/resources/views/components/accordion/item.blade.php

Data Properties / Attributes

Kita juga dapat menentukan atribut mana yang harus dianggap sebagai variabel data menggunakan @props di bagian atas template Blade. Jika ingin memberikan nilai default pada variabel data, kita dapat menentukan nama variabel sebagai kunci array dan nilai default sebagai nilai array: 

   <!-- /resources/views/components/alert.blade.php -->

   @props(['type' => 'info', 'message'])

   <div {{ $attributes->merge(['class' => 'alert alert-'.$type]) }}>
       {{ $message }}
   </div>

Mengingat definisi komponen di atas, bisa komponen seperti ini: 

   <x-alert type="error" :message="$message" class="mb-4"/>

Accessing Parent Data

Terkadang kita ingin mengakses data dari komponen induk di dalam komponen turunan. Dalam kasus ini, kita dapat menggunakan @aware, Misalnya kita sedang membangun komponen menu kompleks yang terdiri dari parent <x-menu> dan child <x-menu.item>: 

   <x-menu color="purple">
     <x-menu.item>...</x-menu.item>
     <x-menu.item>...</x-menu.item>
   </x-menu>

Komponen <x-menu> mungkin memiliki implementasi seperti berikut: 

   <!-- /resources/views/components/menu/index.blade.php -->

   @props(['color' => 'gray'])

   <ul {{ $attributes->merge(['class' => 'bg-'.$color.'-200']) }}>
       {{ $slot }}
   </ul>

Karena colorprop hanya diteruskan ke parent (<x-menu>), prop tidak akan tersedia di dalam <x-menu.item>. Namun, jika kita menggunakan @aware, kita juga dapat membuatnya tersedia di dalam <x-menu.item>: 

   <!-- /resources/views/components/menu/item.blade.php -->

   @aware(['color' => 'gray'])

   <li {{ $attributes->merge(['class' => 'text-'.$color.'-800']) }}>
       {{ $slot }}
   </li>

Anonymous Component Namespaces

komponen anonim biasanya ditentukan dengan menempatkan template Blade di dalam resources/views/components, Namun, terkadang kita ingin mendaftarkan jalur komponen anonim lainnya dengan Laravel selain jalur default.


Untuk memberi tahu Laravel tentang lokasi komponen anonim, kita dapat menggunakan anonymousComponentNamespace yang disediakan oleh Blade fasad. Metode anonymousComponentNamespace menerima "jalur" ke lokasi komponen anonim sebagai argumen pertama dan "ruang nama" tempat komponen harus ditempatkan sebagai argumen kedua. 

   /**
   * Bootstrap any application services.
   *
   * @return void
   */
   public function boot()
   {
       Blade::anonymousComponentNamespace('flights.bookings.components', 'flights');
   }

Dengan contoh di atas, kita dapat merender panel komponen yang ada di dalam direktori komponen yang baru didaftarkan seperti: 

   <x-flights::panel :flight="$flight" />

Building Layout

Layout Using Component

Aplikasi web umumnya memiliki layout atau tata letak yang sama di berbagai halaman. Dengan adanya components blade sangat memudahkan untuk mengatur layout di aplikasi web kita.

# Defining the Layout Component

Contohnya kita membuat aplikasi daftar todo dengan layout komponen seperti kode dibawah : 

< html > 
     < head > 
        < title >{{ $title ?? 'Todo Manager' }}< / title >
    < / head>
    < body >
        < h1 >Todos< / h1 >
        < hr / >
        {{ $slot }}
    < / body >
< / html >

# Applying the Layout Components

Setelah menentukan layout components, kita dapat membuat tampilan blade yang menggunakan komponen tersebut 

< x-layout >
     < li >Makan< / li >
     < li >Belajar< / li >
< / x-layout >

Layout Using Template Inheritence

# Defining the Layout Component

Layout juga dapat dibuat melalui “template inheritance”. Pada layout jenis ini menggunakan @section untuk mendefinisikan bagian konten dan @yield untuk menampilkan konten bagian tertentu. Contoh layout ini seperti kode dibawah : 

< html > 
     < head > 
        < title >App Name - @yield('title')< / title >
    < / head>
    < body >
        @section('sidebar')
           This is the master sidebar.
        @show
        < div class = " container " >
        @yield('content')
        < /div>
    < /body >
< /html >

# Extending a layout

Untuk mendefinisikan tampilan childnya, kita bisa menggunakan @extends parentnya untuk menentukan layout yang diwarisi oleh tampilan childnya. Kita juga dapat menggunakan @section untuk menambahkan konten ke bagian layout. Selanjutnya, akan ditampilkan ke layout dengan menggunakan @yield. Contoh kodenya : 

@extends('layouts.app')
@section('title', 'Page Title')
@section('sidebar')
   @parent
   < p >This is appended to the master sidebar.< / p >
@endsection
@section('content')
   < p >This is my body content.< / p >
@endsection

Form

CSRF field

Kita bisa menggunakan directive @csrf untuk generate hidden field token pada form HTML. 

< form method="POST" action="/profile">
  @csrf 
  .....
< /form>

Method Field

Agar HTML dapat membuat PUT, PATCH, atau DELETE request, Kita perlu menambahkan @method seperti kode dibawah : 

< form action="/foo/bar" method="POST">
  @method('PUT') 
  .....
< /form>

Validation Error

Kita dapat menggunakan @error untuk memeriksa dengan cepat apakah ada pesan kesalahan validasi untuk atribut tertentu. Dalam @erorr arahan, Kita dapat menggunakan $message untuk menampilkan pesan kesalahan. 

< label for="email">Email address < /label> 
< input id="email"
  type="email"
   class="@error('email', 'login') is-invalid @enderror">
@error('email', 'login')
    < div class="alert alert-danger">{{ $message }}< /div>
@enderror
