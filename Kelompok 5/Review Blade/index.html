Introduction

Apa itu Blade ?

Blade adalah template engine yang disediakan Laravel untuk proses templating pada laravel dan sangat bermanfaat dalam proses pengembangan tampilan halaman website. Tidak seperti fitur templating PHP populer lainnya, Blade tidak membatasi developer untuk menggunakan kode PHP biasa dalam membuat kode untuk tampilan. Semua tampilan Blade dikompilasi ke dalam kode PHP biasa dan kemudian disimpan dalam cache hingga diubah, yang berarti Blade pada dasarnya tidak menambahkan beban atau overhead pada saat aplikasi dijalankan. Blade pada laravel menggunakan ekstensi file .blade.php dan biasanya disimpan di direktori resources / views. Dalam hal ini Blade pada Laravel menggunakan basis template inheritance dan sections.


Route::get('/', function () {
return view('greeting', ['name' => 'Fauzi']);
});

Menampilkan Data

Kita dapat menampilkan data yang diteruskan ke tampilan Blade dengan membungkus variabel dalam kurung kurawal.

Route::get('/', function (){

});

Isi variabel dapat ditampilkan sebagai berikut


Hello, {{ $name }}.

namespace App\Providers;
                        
use Illuminate\Support\Facades\Blade;

{
    /**

Blade & JavaScript Frameworks

Kerangka Kerja Blade & JavaScript

Banyak kerangka kerja Javascript menggunakan kurung kurawal, untuk menunjukkan ekspresi tertentu harus ditampilkan di browser dapat menggunakan simbol untuk memberi tahu mesin Blade bahwa ekspresi harus tetap tidak tersentuh. 


<h1>Laravel</h1>
                        
Hello, @{{ name }}.

Dalam contoh ini, simbol akan dihapus oleh blade. Namun, ekspresi akan tetap tidak tersentuh oleh mesin Blade, memungkinkan untuk dirender oleh kerangka JavaScript. @{{ name }}

Simbol juga dapat digunakan untuk menghindari arahan Blade


{{-- Blade template --}}
@@if()
                        
<!-- HTML output -->
@if()

Merender JSON

Pada saat meneruskan arrya ke tammpilan dengan tujuan merendernya sebagai JSON, kita dapat menggunakan metode direktif daripada memanggilnya secara manual. Metode ini menerima argumen yang sama seperti fungsi PHP. Akan tetapi, metode ini akan memastikan bahwa JSON yang dihasilkan lolos dengan benar untuk dimasukkan dalam kutipan HTML. Metode ini akan mengembalikan pernyataan JavaScript string yang akan mengubah objek atau array yang diberikan menjadi objek JavaScript yang valid. json_encode Illuminate\Support\Js::from from json_encode from JSON.parse 


<script>
   var app = {{ Illuminate\Support\Js::from($array) }}
</script>

Versi terbaru kerangka aplikasi Laravel menyertakan fasad, yang menyediakan akses mudah ke fungsi di dalam template blade 

<script>
   var app = {{ Js::from($array) }}
</script>

The @once Directive

@once Directive

@once Directive memungkinkan kita untuk menentukan bagian dari template yang hanya akan dievaluasi sekali per siklus rendering. Ini mungkin berguna untuk menjalankan dahulu bagian tertentu dari JavaScript ke dalam header halaman menggunakan stacks. Seperti contoh, jika kita merender komponen tertentu ke dalam suatu perulangan, kita mungkin hanya ingin mendorong JavaScript ke header saat pertama kali komponen dirender :


   @once
       @push('scripts')
           <script>
               // Your custom JavaScript...
           </script>
       @endpush
   @endonce

Karena @oncedirektif sering digunakan bersama dengan @pushor @prependdirektif, @pushOncedan @prependOncedirektif tersedia untuk kenyamanan Anda :


   @pushOnce('scripts')
       <script>
          // Your custom JavaScript...
       </script>
   @endPushOnce

Raw PHP

Raw

Di beberapa situasi, Raw sangat berguna untuk menyematkan kode PHP sangat berguna ke dalam tampilan kita. Kita dapat menggunakan direktif Blade @php untuk mengeksekusi blok PHP biasa di dalam template kita. Jika dalam single statement, statement dapat dimasukkan atau diikutkan didalam direktif @php.


   @php
       $counter = 1;
   @endphp

Kita hanya perlu menulis satu pernyataan PHP untuk dapat memanggil statement yang telah kita buat di Raw PHP, seperti contoh code yang ada di bawah :


 1@php($counter = 1)
