Introduction

Apa itu Blade ?

Blade adalah template engine yang disediakan Laravel untuk proses templating pada laravel dan sangat bermanfaat dalam proses pengembangan tampilan halaman website. Tidak seperti fitur templating PHP populer lainnya, Blade tidak membatasi developer untuk menggunakan kode PHP biasa dalam membuat kode untuk tampilan. Semua tampilan Blade dikompilasi ke dalam kode PHP biasa dan kemudian disimpan dalam cache hingga diubah, yang berarti Blade pada dasarnya tidak menambahkan beban atau overhead pada saat aplikasi dijalankan. Blade pada laravel menggunakan ekstensi file .blade.php dan biasanya disimpan di direktori resources / views. Dalam hal ini Blade pada Laravel menggunakan basis template inheritance dan sections.


Route::get('/', function () {
return view('greeting', ['name' => 'Fauzi']);
});

Merender JSON

Pada saat meneruskan arrya ke tammpilan dengan tujuan merendernya sebagai JSON, kita dapat menggunakan metode direktif daripada memanggilnya secara manual. Metode ini menerima argumen yang sama seperti fungsi PHP. Akan tetapi, metode ini akan memastikan bahwa JSON yang dihasilkan lolos dengan benar untuk dimasukkan dalam kutipan HTML. Metode ini akan mengembalikan pernyataan JavaScript string yang akan mengubah objek atau array yang diberikan menjadi objek JavaScript yang valid. json_encode Illuminate\Support\Js::from from json_encode from JSON.parse 


<script>
   var app = {{ Illuminate\Support\Js::from($array) }}
</script>

Versi terbaru kerangka aplikasi Laravel menyertakan fasad, yang menyediakan akses mudah ke fungsi di dalam template blade 

<script>

The @once Directive

@once Directive

@once Directive memungkinkan kita untuk menentukan bagian dari template yang hanya akan dievaluasi sekali per siklus rendering. Ini mungkin berguna untuk menjalankan dahulu bagian tertentu dari JavaScript ke dalam header halaman menggunakan stacks. Seperti contoh, jika kita merender komponen tertentu ke dalam suatu perulangan, kita mungkin hanya ingin mendorong JavaScript ke header saat pertama kali komponen dirender :


   @once
       @push('scripts')
           <script>
               // Your custom JavaScript...
           </script>
       @endpush
   @endonce

Karena @oncedirektif sering digunakan bersama dengan @pushor @prependdirektif, @pushOncedan @prependOncedirektif tersedia untuk kenyamanan Anda :


   @pushOnce('scripts')
       <script>
          // Your custom JavaScript...
       </script>
   @endPushOnce

Raw PHP

Raw

Di beberapa situasi, Raw sangat berguna untuk menyematkan kode PHP sangat berguna ke dalam tampilan kita. Kita dapat menggunakan direktif Blade @php untuk mengeksekusi blok PHP biasa di dalam template kita. Jika dalam single statement, statement dapat dimasukkan atau diikutkan didalam direktif @php.

       $counter = 1;
   @endphp

Kita hanya perlu menulis satu pernyataan PHP untuk dapat memanggil statement yang telah kita buat di Raw PHP, seperti contoh code yang ada di bawah :

Data Properties / Attributes

Properti / Atribut Data

Kita dapat menentukan atribut yang harus dianggap sebagai variabel data menggunakan direktif di bagian atas template Blade. Semua atribut lain pada komponen akan tersedia melaui tas atribut komponen. Jika ingin memberikan variabel data nilai default, kita dapat menentukan nama variabel sebagai kunci array dan nilai default sebagai array. 


<!-- /resources/views/components/alert.blade.php -->                            
                        
                        
@props(['type' => 'info', 'message'])

                        
<div {{ $attributes->merge(['class' => 'alert alert-'.$type]) }}>
   {{ $message }}
</div>

Kita dapat merender komponen seperti berikut


<x-alert type="eror" :message="$message" class="mb-4"/>

Anonymous Components Namespaces

Namespace Komponen Anonim

Komponen anonim biasanya didefinisikan dengan menempatkan template Blade di dalam direktori. Terkadang ingin mendaftarkan jalur komponen anonim alainnya dengan Laravel selain jalur default.

Laravel mempunayi metode yang disediakan oleh fasad. Metode ini menerima "jalur" ke lokasi komponen anonim sebagai argumen pertama dan "namespace yang harus ditempatkan di bawah komponen sebagai argumen kedua." 


/**
 * Bootstrap any application service
 *
 * @return void
 */
public function boot()
{
   Blade::anonymousComponentNamespace('flights.bookings.components', 'flights');
}

Kita dapat merender komponen seperti berikut


<x-flights::panel :flight="$flight" />

Layouts Using Components

Layouts Using Components

Sebagian besar aplikasi web mempertahankan tata letak umum yang sama di berbagai halaman. Akan sangat merepotkan dan sulit untuk memelihara aplikasi jika harus mengulang seluruh tata letak HTML di setiap tampilan yang di buat. Untungnya, lebih mudah untuk mendefinisikan tata letak ini sebagai satu komponen Blade dan kemudian menggunakannya di seluruh tampilan.Misalnya, bayangkan kita sedang membangun aplikasi daftar "todo". Kita mungkin mendefinisikan layoutkomponen yang terlihat seperti berikut:


Defining The Layout Component

Applying The Layout Component

Setelah layout komponen telah ditentukan, kita dapat membuat tampilan Blade yang menggunakan komponen tersebut. Seperti contoh di bawah ini, kita akan mendefinisikan tampilan sederhana yang menampilkan daftar tugas kita: 


<x-layout>
@foreach ($tasks as $task)
{{ $task }}
@endforeach
</x-layout>

Ingat, konten yang disuntikkan ke dalam komponen akan diberikan ke variabel $slot default di dalam layout komponen kita. Seperti yang mungkin kita perhatikan, layout kita juga menghargai $titleslot jika tersedia; jika tidak, judul default akan ditampilkan. 


<x-layout>
<x-slot:title>
  Custom Title
</x-layout>
@foreach ($tasks as $task)
{{ $task }}
@endforeach
</x-layout>

Setelah kita mendefinisikan tampilan tata letak dan daftar tugas, kita hanya perlu mengembalikan tasktampilan dari rute: 


use App\Models\Task;
Route::get('/tasks', function () {
return view('tasks', ['tasks' => Task::all()]);
});

Validation Errors

Validation Errors

Notasi @error dapat digunakan untuk memeriksa dengan cepat apakah ada pesan kesalahan validasi untuk atribut tertentu. Dalam notasi @error, kita dapat menggunakan $message variabel untuk menampilkan pesan kesalahan. 


<label for="title">Post Title
<input id="title"
type="text"
class="@error('title') is-invalid @enderror">
@error('title')
<div class="alert alert-danger">{{ $message }}
@enderror

Kita juga dapat menambahkan notasi @else untuk menampilkan tampilan jika tidak error dan diakhiri dengan notasi @enderror.


<label for="email">Email address
<input id="email"
type="email"
class="@error('email') is-invalid @else is-valid @enderror">
